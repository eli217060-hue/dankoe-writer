## Thread 写作模式总结

### 1. 钩子的六种类型

| 类型 | 公式 | 示例 |
|------|------|------|
| 利益承诺 | "How to [X] [N]x faster" | 10x faster learning |
| 隐藏资产 | "You have [价值] trapped in [地方]" | $100K in your head |
| 负面转正 | "The best [结果] came after [负面]" | fed up → breakthrough |
| 权威引用 | "[名言]" + thread on [应用] | Naval + intelligence |
| 否定主流 | "[主流建议] is wrong" | "Niche down" |
| 痛点堆叠 | "No X. No Y. No Z." | writing sucks |

### 2. Thread 的通用结构

```
推文1: 钩子（必须独立成立）
推文2: 扩展钩子或定义核心概念
推文3-5: 问题诊断 / 为什么你的旧方法不行
推文6-15: 核心内容（方法/框架/步骤）
推文16-17: 总结 + 行动号召
最后一条: 金句或反转
```

### 3. 每条推文的独立价值原则

Dan Koe 的 Thread 中，每条推文都可以单独发布：
- 即使只看一条也有价值
- 这是短内容的核心法则
- 也是为什么他的 Thread 能被大量转发

### 4. 数字的威力

Dan Koe 高频使用数字：
- $100,000、$4.1M、10x、2-4 hours
- 数字创造具体性和可信度
- 数字让标题和钩子更有点击欲
