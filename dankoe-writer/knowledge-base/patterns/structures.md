# 结构模式

> 从 Dan Koe 文章中提取的文章结构，与 SOP 库配合使用
> 新增：结构选择决策树（Step 10b 使用）

---

## 结构选择决策树

Step 10b 选择结构时，按以下逻辑匹配：

```
主题类型判断
│
├─ 主题偏认知/心理（拖延、焦虑、身份、思维方式）
│  ├─ 需要深度剖析 → 层层深入型
│  └─ 需要快速打动 → 微故事型
│
├─ 主题偏行动/方法（如何做X、提高Y、解决Z）
│  └─ → 问题→分析→方案型
│
├─ 主题偏趋势/时代（AI、行业变化、未来预测）
│  ├─ 有历史维度 → 史诗演化型
│  └─ 需要多层展开 → 维度递进型
│
├─ 主题偏经验/规则（多年经验总结、人生原则）
│  └─ → 规则列表型
│
├─ 主题偏情感/故事（个人转变、客户案例、共鸣向）
│  └─ → 故事先行型
│
└─ 不确定 / 兼有多种
   └─ → 对比论证型（最通用，适合大多数主题）
```

**篇幅约束**：
| 结构 | 短文 | 中文 | 长文 |
|------|------|------|------|
| 层层深入型 | ❌ | ✅ | ✅ |
| 微故事型 | ✅ | ✅ | ✅（作为开头） |
| 问题→分析→方案型 | ❌ | ✅ | ✅ |
| 规则列表型 | ❌ | ✅ | ✅ |
| 对比论证型 | ❌ | ✅ | ✅ |
| 故事先行型 | ✅（3句话版） | ✅ | ✅ |
| 维度递进型 | ❌ | ❌ | ✅ |
| 史诗演化型 | ❌ | ❌ | ✅ |
| 短文专用结构 | ✅ | ❌ | ❌ |

## 一、层层深入型

```
表面现象 → 原因分析 → 心理层面 → 解决方案 → 行动召唤
```

**Dan Koe 用法**：不在第一层停下来。大多数作者写到"原因分析"就给方法论了，Dan Koe 会多挖一层到心理机制，让读者觉得"别人只告诉我 what，他告诉我 why behind why"。

**示例文章**：The Unpleasant Truth: AI Is Coming For Your Job
- 表面：AI 要取代你的工作
- 原因：平庸工作可以被自动化
- 心理：你的独特性是什么？
- 方案：构建不可替代的技能组合
- 行动：4 个具体步骤

**适合篇幅**：中文、长文
**适合主题**：需要深度剖析的认知/心理类主题

**常见错误**：
- ❌ 每层都差不多长 → 应该"心理层面"占最大比重
- ❌ 直接从"表面"跳到"方案" → 中间的分析层是这个结构的灵魂

---

## 二、规则列表型

```
Rule 1: [核心主张] → [解释]
Rule 2: [核心主张] → [解释]
...
```

**Dan Koe 用法**：规则之间有递进关系，不是并列罗列。前面的规则是后面的基础，最后的规则是升华。

**示例文章**：12 rules to change your life in 12 months
- 前 4 条：基础认知（目标、身份）
- 中间 4 条：行为改变（习惯、专注）
- 后 4 条：系统构建（创作、复利）

**适合篇幅**：中文、长文
**适合主题**：经验总结、人生原则、方法论集合

**常见错误**：
- ❌ 规则之间完全并列，删掉任何一条都不影响其他 → 应该有递进
- ❌ 每条规则都写 300 字平均用力 → 最重要的 2-3 条多写，其他可以短

---

## 三、对比论证型

```
[主流观点] → [为什么失效] → [新范式] → [具体方法]
```

**Dan Koe 用法**：先承认旧观点有道理（"它曾经有效"），再解释为什么现在不行了，最后提供替代方案。不是简单地"你错了我对了"。

**示例文章**：The death of value-based content
- 旧：价值型内容有效
- 为什么失效：每个人都在做，AI 也会做
- 新范式：视角型内容
- 方法：如何构建独特视角

**适合篇幅**：中文、长文
**适合主题**：最通用的结构，特别适合反共识类主题

**常见错误**：
- ❌ 上来就否定旧观点 → 应该先承认它"曾经有效"，这样转折更有力
- ❌ 新范式只是旧范式的反面 → 应该比旧范式高一个维度

## 四、问题→分析→方案型

```
[痛点问题] → [深度分析为什么] → [解决框架] → [行动步骤]
```

**Dan Koe 用法**：分析部分占文章最大比重（40-50%），因为 Dan Koe 认为"理解问题比解决问题更重要"。

**示例文章**：How to fix your entire life in 1 day
- 痛点：人生感觉一团糟
- 分析：目标镜头理论 + 身份-行为模型
- 框架：一天重构法
- 步骤：具体的日程安排

**适合篇幅**：中文、长文
**适合主题**：方法论分享、实操指南

**常见错误**：
- ❌ 问题只用一段话带过就急着给方案 → 分析部分应该是最厚的
- ❌ 行动步骤太抽象 → 具体到读者读完就能做

## 五、故事先行型

```
[个人故事/场景] → [从故事中提炼洞察] → [扩展到普遍原理] → [读者可以做什么]
```

**Dan Koe 用法**：故事不是开胃菜，而是证据。故事里已经包含了核心论点的种子。

**适合篇幅**：所有篇幅（短文用 3 句话版本，长文用完整叙事）
**适合主题**：个人转变、客户案例、共鸣向内容

**常见错误**：
- ❌ 故事和后面的论点没有关系 → 故事本身应该是论点的具象化
- ❌ 故事太长，论点太短 → 故事和论点的篇幅应该大致相当

## 六、微故事型（Dan Koe 自述最常用的快速起手式）

Problem（问题）→ Amplify（放大）→ Solution（解决方案）

**来源**：Dan Koe 在 "How to Articulate Yourself Intelligently" 中说："After 6 years of doing this, it's still my go-to when I need a way to articulate a thought fast."

**与 APAG 的区别**：
- APAG 是完整文章的骨架（Attention → Perspective → Advantage → Gamify）
- 微故事法是**最小可用写作单元**，一段话、一条短帖、一个段落内部都能用
- APAG 的 Attention 部分本身常常就是一个微故事

**三种使用方式**：

短帖版（3-5 句话）：
- 1 句话点出读者认同的问题
- 1-2 句话说如果不解决会怎样
- 1-2 句话给出解决方案或重构视角

段落版（文章内部某个段落的结构）：
- 问题描述，让读者点头
- 放大：继续不解决的后果
- 转折：但如果换个角度看…
- 解决方案

长文版（作为开头钩子）：
- 问题 + 放大 = 前 300-500 字的钩子
- 后续内容 = 展开解决方案

**适合篇幅**：所有篇幅（核心优势 — 可伸缩）
**与 SOP 的关系**：不对应特定 SOP，可嵌入任何 SOP 的任何段落中

## 七、维度递进型

```
维度 1（最简单）→ 解释
维度 2（增加深度）→ 解释
维度 3（增加高度）→ 解释
...逐层递进
```

**Dan Koe 用法**：5D thinking 文章，从线→层级→高度→四象限→历史，文章结构本身就在示范核心概念。

**适合篇幅**：长文
**适合主题**：需要多层展开的认知类主题

**常见错误**：
- ❌ 维度之间没有明确的递进关系 → 每个维度必须比前一个"高一层"
- ❌ 读者到第三个维度就失去耐心 → 每个维度的开头需要重新激发好奇心

---

## 八、史诗演化型

```
历史阶段 1 → 当时的特征
历史阶段 2 → 当时的特征
历史阶段 3 → 当时的特征
当前危机 → 问题所在
下一阶段 → 你的行动
```

**Dan Koe 用法**：Future of work 文章，从采集社会写到 AI 时代。

**适合篇幅**：长文
**适合主题**：时代趋势、行业演化、技术变革

**常见错误**：
- ❌ 历史部分写太长，读者等不到"跟我有什么关系" → 历史铺垫控制在文章前 30%
- ❌ "下一阶段"写得太空泛 → 必须落到读者具体能做的事

## 短文专用结构

短文不使用上述完整结构，而是用以下简化版：

### 结构 A：钩子 + 洞察（认知型）
```
[反常识钩子] → [核心洞察，1-2 句话展开] → [结尾留白]
```

### 结构 B：场景 + 点睛（情感型）
```
[3 句话场景] → [类比或金句点睛]
```

### 结构 C：洞察 + 行动（实用型）
```
[核心洞察] → [1 个立即可做的行动] → [开放式问题收尾]
```

**参见 `short-form.md` 获取更详细的短内容规则。**

## 结构与 SOP 的关系

| 结构 | 对应 SOP |
|------|----------|
| 对比论证型 | SOP-001 反共识型 |
| 问题→分析→方案型 | SOP-002 方法论型 |
| 规则列表型 | SOP-003 规则列表型 |
| 层层深入型 | 可用于多种 SOP |
| 故事先行型 | 可用于多种 SOP |
| 微故事型 | 不对应 SOP，嵌入任何结构 |
| 维度递进型 | 可用于多种 SOP |
| 史诗演化型 | 可用于多种 SOP |
| 短文专用结构 | 根据积木组合动态选择 |
