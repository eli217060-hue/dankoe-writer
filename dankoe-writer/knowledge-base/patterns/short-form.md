# 短内容写作模式

> Dan Koe 短帖的写作逻辑——短文不是压缩版长文，是完全不同的思维方式

---

## 一、核心原则：短文和长文是不同物种

| 维度 | 短文 | 长文 |
|------|------|------|
| 观点数量 | 1 个观点打穿 | 多层递进 |
| 论证方式 | 类比即论证，不额外解释 | 需要铺垫、数据、展开 |
| 节奏 | 短句跳跃，留白让读者思考 | 流畅过渡，引导读者 |
| 结尾 | 开放式，不总结 | 总结 + CTA |
| 故事 | 最多 3 句话的场景闪回 | 完整叙事弧 |
| 行动建议 | 最多 1 个，3 步以内 | 可以是完整方法论 |
| 过渡句 | 不需要，跳跃是节奏 | 需要，流畅是节奏 |

**关键心态转换**：
写长文时你是老师在讲课；写短文时你是朋友在说一句让人停下来想的话。

## 二、Dan Koe 短帖的 4 个特征

### 特征 1：一个观点打穿，不展开

Dan Koe 的短帖从不试图"说完整个事情"。一个洞察就是全部内容。

```
做法：
- 选积木库中最强的 1 块核心洞察
- 不解释为什么，不给方法论
- 让读者自己去想

反面：
- 不要"首先...其次...最后..."
- 不要"让我解释一下为什么"
- 不要试图在 500 字里塞入完整论证
```

### 特征 2：用节奏感代替论证

短句。
断句。
每句话换行。
让空白替你说话。

```
做法：
- 一句一段
- 句子之间靠逻辑跳跃连接，不靠过渡句
- 最重要的句子单独成段，前后留空

反面：
- 不要写连续 3 行以上的长段落
- 不要用"因此""所以""综上"这些连接词
- 不要"让我来分析一下"
```

### 特征 3：类比即论证

长文里类比是辅助工具，需要配合解释。
短文里类比本身就是全部的解释。

```
做法：
- 直接给出类比，不加"换句话说"
- 类比要足够生动，读者一看就懂
- 一个好类比顶三段论证

示例：
✓ "你的大脑不是电脑，你不能 ctrl+alt+del 强制关机。"
✗ "大脑的工作机制类似于电脑。就像电脑一样，你不能通过强制关机的方式来让大脑休息。这是因为..."
```

### 特征 4：结尾开放，不总结

长文结尾是："所以你应该做这三步..."
短文结尾是："问题是——你的'关机信号'是什么？"

```
做法：
- 用问题收尾（让读者自己思考）
- 或用金句收尾（留下回响）
- 最后一句的力量 > 前面所有内容

反面：
- 不要"总之""综上所述""最后想说"
- 不要重复开头的观点
- 不要给出完整的行动清单
```

## 三、短文的 3 种组装结构

### 结构 A：钩子 + 洞察（认知型）

```
[反常识钩子，1-2 句]
[核心洞察，展开但不超过 3 段]
[金句或开放问题收尾]
```

适合：挑战常识、提供新视角
积木组合：反常识钩子 + 核心洞察 + 金句

### 结构 B：场景 + 点睛（情感型）

```
[3 句话画面场景]
[类比或金句，一句话点破本质]
```

适合：引发共鸣、场景代入
积木组合：故事（场景版）+ 类比或金句

### 结构 C：洞察 + 行动（实用型）

```
[核心洞察，直接给出]
[1 个行动步骤，不超过 3 步]
[开放式问题收尾]
```

适合：给出可执行建议
积木组合：核心洞察 + 行动步骤 + 问题

## 四、平台适配规则

### 小红书

```
字数：300-600 字
段落：每段不超过 3 行
emoji：可以用但不超过 8 个，不要每段都有
格式：可以用「」强调关键词，避免大段 bullet point
标题：加数字，控制在 20 字内
图片：如果是图文帖，关键信息放在图片里
```

### Twitter/X

```
字数：单条 280 字符以内，Thread 5-7 条
每条推文自成一体（即使只读一条也有价值）
Thread 第一条是钩子，最后一条是金句
中间每条都是一个独立积木
```

### 即刻

```
字数：200-500 字
语气：比小红书更随意，像朋友说话
可以用更多口语化表达
标签要精准
```

## 五、从同一批积木拼出不同篇幅的示例

**主题**：拖延症
**选定角度**：拖延不是懒，是大脑在保护你

### 短文版（小红书，400 字）

用积木：反常识钩子 #2 + 核心洞察 #1 + 问题 #1

```
你不是拖延。

你的大脑在保护你。

每次你"拖延"，其实是大脑在说：
"这件事做了可能会失败，失败了会被评判，所以——别做了。"

它不是在偷懒，是在当你的保镖。
只是这个保镖过于紧张了。

问题不是"怎么克服拖延"。
问题是：你在害怕什么评判？
```

### 长文版（公众号，2000 字）

用积木：钩子 #3 + 洞察 #1 + 故事 #2 + 数据 #1 + 类比 #2 + 行动步骤 #1+#3 + 金句 #4

```
[完整展开：问题诊断开场 → 个人故事 → 心理学研究数据 →
 身份-行为模型分析 → 具体的身份转变方法 → 金句收尾]
```

**注意区别**：
- 短文只用了 3 块积木，长文用了 8 块
- 短文没有过渡句，长文有完整的逻辑链
- 短文以问题结尾（开放），长文以金句 + 行动结尾（收束）
- 两篇文章读起来完全不像同一篇的不同版本
