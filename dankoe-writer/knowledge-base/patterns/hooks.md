# 开场钩子模式

> 从 Dan Koe 文章中提取的 7 种开场模式
> 每种模式附带正面示例和**反面示例**，帮助 AI 识别"符合模式但不好"的钩子

---

## 模式 1：数据冲击

```
[惊人数据].
And that's okay.
Most people do.
```

**机制**：用一个出乎意料的数字或事实开场，读者的第一反应是"真的吗？"，然后继续读。

**Dan Koe 用法**：通常配合"And that's okay"来缓冲冲击，暗示"别慌，我有解法"。

**✅ 好的示例**：
- "80% 的新年决心在 2 月前就失败了。And that's okay. Most people are setting the wrong goals."
- "京东数据显示芳疗产品销量翻了 5 倍。但央视调查发现大部分精油是掺假的。"
- "过去三个月，我花了 147 个小时'准备开始'。真正做事的时间？11 个小时。"

**❌ 坏的示例**：
- "研究表明，大多数人都有拖延的习惯。"——太温吞，没有冲击感，"研究表明"是最弱的开头
- "据统计，很多创业者在第一年就失败了。"——"据统计""很多"都是模糊词，不如给具体数字
- "数据显示，焦虑症患者逐年增加。"——正确但无聊，读者看过一百遍了

**判断标准**：数据要么让人"卧槽"，要么别用。如果数据只是"嗯对，我知道"，就不是冲击。

## 模式 2：权威引用

```
> [名人名言]

This quote changed everything for me.
Here's why...
```

**机制**：借权威人物的话增加可信度，但快速转向自己的分析框架——权威只是钩子，不是核心。

**✅ 好的示例**：
- "Fiverr CEO 的一条消息震动了整个自由职业圈。但大多数人误读了它。"
- "张一鸣 2012 年的一条微博，预言了今天所有人的困境。"

**❌ 坏的示例**：
- "'成功就是从失败走到失败，而不丧失热情。'——丘吉尔。今天我想聊聊成功。"——引用太常见，过渡太无聊
- "马云曾经说过……"——用马云、乔布斯、马斯克的引用已经是内容创作的月经贴
- "'活在当下。'——某位哲学家。"——来源模糊，读者不信

**判断标准**：权威引用后面必须紧跟"但大多数人没注意到的是"或"他说的其实不是这个意思"——引用只是起跳板，落点必须是你的独特解读。

## 模式 3：问题诊断

```
You're probably going to [失败行为].
Here's the real reason why...
```

**机制**：指出读者正在犯的错误（但不指责），制造"我被看穿了"的感觉。

**✅ 好的示例**：
- "你今晚可能又会'努力入睡'。这恰恰是你失眠的原因。"
- "你大概率正在做一件事：用战术上的勤奋掩盖战略上的懒惰。"

**❌ 坏的示例**：
- "你是不是也经常拖延？"——太泛，每个人都拖延，没有诊断感
- "你有没有发现自己很焦虑？"——读者会想"废话"，然后划走
- "很多人都有这个问题……"——"很多人"不是诊断，是统计。诊断应该让读者觉得"你怎么知道的"

**判断标准**：好的诊断 = 描述一个具体行为 + 揭示这个行为背后的反直觉原因。如果只有"你是不是也……"就不是诊断，是问卷调查。

## 模式 4：反直觉陈述

```
[主流观点] is wrong.
Here's what actually works...
```

**机制**：直接否定大家深信不疑的东西，制造强烈的认知冲突。

**✅ 好的示例**：
- "Value-based content is dead. Not because it doesn't work — but because everyone is doing it."
- "'找到你的热情'是最糟糕的建议。热情不是找到的，是创造的。"
- "自律是个骗局。真正改变你的不是意志力，是环境设计。"

**❌ 坏的示例**：
- "你以为的 X 其实是错的！"——空洞，没说错在哪里，只是在喊口号
- "颠覆你的认知！"——这不是反直觉陈述，这是标题党
- "XX 已死。"——如果后面只是"所以我们要做 YY"，没有解释"为什么死了"，就只是哗众取宠

**判断标准**：反直觉的力量不在于否定，在于否定后的解释。"X is wrong because Y"中，Y 的质量决定钩子的质量。

## 模式 5：个人故事

```
I used to believe [旧信念].
Then [转折事件].
Now I understand [新认知]...
```

**机制**：用"我曾经和你一样"拉近距离，用转折制造好奇心。

**✅ 好的示例**：
- "三年前我每天工作 14 小时，以为这就是成功的代价。直到有一天我在公司厕所里哭了 20 分钟。"
- "我曾经以为多兴趣是我的缺陷。直到我发现 Dan Koe、Naval Ravikant 都是通才。"

**❌ 坏的示例**：
- "我曾经也是一个普通人。后来我改变了。"——太模糊，没有具体场景，读者无法代入
- "说起这个话题，我有很多感触。"——这不是故事，这是预告片。直接讲故事
- "作为一个在行业里深耕了十年的人……"——自我背书开头会让读者防御心拉满

**判断标准**：好的故事开头必须有具体的时间、地点或动作。"三年前""公司厕所""哭了20分钟"——这些细节让故事变得真实。

## 模式 6：连续问题轰炸

```
[终极问题1]？
[终极问题2]？
[终极问题3]？
If [条件], what do we do?
```

**机制**：连续抛出无法回避的终极问题，读者被逼到必须继续读。

**✅ 好的示例**：
- "AI 会取代创意工作吗？金钱会过时吗？如果挣扎、地位和好奇心是意义的来源，而 AGI 要消除它们，我们怎么办？"

**❌ 坏的示例**：
- "你有没有想过为什么……你有没有好奇过……你有没有注意到……"——三个"你有没有"不是问题轰炸，是啰嗦
- "什么是成功？什么是幸福？什么是人生的意义？"——问题太大太空，读者会觉得你在写哲学课本而不是文章
- "为什么？为什么？为什么？"——光有"为什么"没有具体指向，不是轰炸，是噪音

**判断标准**：每个问题必须比前一个更尖锐。问题之间要有递进关系，最后一个问题应该是读者"不敢想但不得不想"的那个。

## 模式 7：三类人分类

```
每次 [变革] 出现，三种人浮出水面：

[消极类型]：[一句话描述]
[中间类型]：[一句话描述]
[积极类型]：[一句话描述]
```

**机制**：读者自动对号入座。没人想当前两类人。

**✅ 好的示例**：
- "抗拒者把身份绑在旧方式上。等待者看到变化但告诉自己'会过去的'。好奇者实验、构建、适应。"

**❌ 坏的示例**：
- "有的人成功了，有的人失败了，有的人还在观望。"——没有洞察，只是陈述事实
- "第一种人很聪明，第二种人很努力，第三种人既聪明又努力。"——没有张力，读者不会对号入座
- "普通人 vs 高手 vs 顶级高手"——这不是分类，是等级划分，读者会反感

**判断标准**：三类人的划分标准不是能力高低，而是面对变化的**态度**差异。读者应该在第一类里看到自己的影子，在第三类里看到自己想成为的样子。

## 钩子选择指南

| 文章类型 | 推荐钩子 | 原因 |
|----------|----------|------|
| 反共识型 | 模式 4（反直觉）或模式 1（数据冲击）| 需要快速打破旧认知 |
| 方法论型 | 模式 3（问题诊断）或模式 5（个人故事）| 需要先建立共鸣 |
| 规则列表型 | 模式 1（数据冲击）或模式 2（权威引用）| 需要建立权威感 |
| 趋势预测型 | 模式 6（问题轰炸）或模式 7（三类人）| 需要制造紧迫感 |

| 篇幅 | 钩子处理 |
|------|----------|
| 短文 | 钩子即开头，1-2 句话直接进入 |
| 中文 | 钩子可以展开到 2-3 段 |
| 长文 | 钩子可以是完整的开场故事 |
