# AI 写作反模式库

> 本文件列出 AI 生成中文内容时最常犯的错误模式。
> Step 12 生成初稿时持续检查，任何命中项必须立即重写该句/段。

---

## 一、开头反模式

### 1.1 空洞宏大开头（命中率最高）

```
❌ "在当今社会，越来越多的人开始关注……"
❌ "随着科技/经济/AI 的飞速发展……"
❌ "众所周知……"
❌ "在这个信息爆炸的时代……"
❌ "不知道你有没有过这样的经历……"（伪互动，读者一眼看穿是 AI）

✅ 直接切入痛点场景、数据、反常识观点或微故事
✅ 第一句话就应该让读者停下来
```

**检测规则**：文章前 50 字内出现"当今""随着""众所周知""不知道你" → 立即重写开头。

### 1.2 假装提问开头

```
❌ "你有没有想过，为什么……"（没有悬念感，只有套路感）
❌ "你是否也曾经……"
❌ "如果我告诉你……你会怎么想？"

✅ 如果要用问题开头，问题本身必须包含反常识信息
✅ 好的问题开头示例："为什么越自律的人越容易崩溃？"（问题本身就是钩子）
```

---

## 二、论证反模式

### 2.1 教科书式论证

```
❌ "首先……其次……再次……最后……"
❌ "第一……第二……第三……"（在非列表型文章中）
❌ "一方面……另一方面……"
❌ "从某种意义上来说……"

✅ 用故事、类比、场景切换来推进逻辑
✅ 段落之间靠认知落差连接，不靠连接词
```

**检测规则**：非规则列表型文章中出现"首先/其次/再次/最后"四连 → 必须打散重组。

### 2.2 伪深度论证

```
❌ "这背后的深层原因是……"（然后给出一个并不深的原因）
❌ "本质上来说……"（然后只是换了个说法重复上一段）
❌ "从更宏观的角度来看……"（然后说了一句正确但空洞的废话）
❌ "这不仅仅是 X 的问题，更是 Y 的问题"（Y 和 X 其实是同一件事）

✅ 真正的深度 = 说出读者没想到但一听就觉得对的东西
✅ 如果你要说"本质上"，后面那句话必须能让读者停顿 3 秒
```

**检测规则**：出现"深层原因""本质上""更宏观"后，检查下一句是否真的比上一段深一层。如果只是换了措辞 → 删掉这个"深度标签"，直接说事。

### 2.3 万能连接词滥用

```
❌ "然而""因此""综上所述""值得注意的是""不可否认"
❌ "诚然……但是……"（AI 最爱的让步转折套路）
❌ "换句话说……"（如果需要换句话说，说明第一句没说清楚）

✅ 好的过渡是靠内容本身的逻辑跳跃完成的
✅ 如果删掉连接词后段落依然通顺 → 删掉它
```

**检测规则**：每 500 字中"然而/因此/综上/诚然/值得注意的是"出现超过 3 次 → 逐一检查能否删除。

## 三、表达反模式

### 3.1 翻译腔

```
❌ "那些在深夜里独自面对着屏幕上闪烁的代码的年轻程序员们"（超长定语从句）
❌ "这是一个关于如何在不确定性中找到确定性的故事"
❌ "你不是不够努力，你是还没有成为那个会在那里的人"（直译英文结构）

✅ 中文用短句。主谓宾，干净利落
✅ 把长定语拆成两个短句
✅ "凌晨两点，他盯着屏幕上飘红的代码" — 动作先行，场景自现
```

**检测规则**：
- 单句超过 40 字且只有一个主语 → 拆分
- "那些……的人们" 结构 → 重写
- "这是一个关于……的……" 结构 → 重写

### 3.2 情绪注水

```
❌ "这真的非常非常重要"
❌ "说实话，我真的觉得……"
❌ "不得不说……"
❌ "我想说的是……"（直接说就行了）
❌ 连续 3 个以上感叹号 "!!!"

✅ 情绪靠事实和场景传递，不靠副词和语气词堆砌
✅ 如果你要强调，用短句 + 独立成段，而不是加"非常"
```

### 3.3 假口语化

```
❌ "说白了就是……"（每篇都用就不白了）
❌ "简单来说……"（然后说了一段不简单的话）
❌ "你品，你细品"（2020 年的梗了）
❌ "懂的都懂"
❌ "家人们"（除非真的在小红书写种草文）

✅ 真正的口语化是句子结构简单，不是塞网络流行语
✅ 测试方法：这句话你会当面对朋友说吗？不会 → 改
```

## 四、结尾反模式

### 4.1 教师总结式结尾

```
❌ "总之……"
❌ "综上所述……"
❌ "希望这篇文章对你有所启发/帮助"
❌ "让我们一起……吧！"
❌ "以上就是关于 X 的全部内容"

✅ 用金句收尾（让读者带走一句话）
✅ 用开放问题收尾（让读者继续想）
✅ 用行动号召收尾（让读者立刻做一件事）
✅ 最好的结尾是：删掉最后一段，文章反而更有力
```

**检测规则**：文章最后 100 字出现"总之/综上/希望/让我们一起" → 立即重写结尾。

### 4.2 强行升华

```
❌ "愿我们都能成为更好的自己"
❌ "人生没有白走的路，每一步都算数"
❌ "在这个不确定的时代，唯一确定的是……"
❌ "或许，这就是生活的意义吧"

✅ 结尾的力量来自前文的铺垫，不是最后硬拔高度
✅ 如果前文没有铺垫这个高度 → 结尾就不该到这个高度
```

## 五、结构反模式

### 5.1 对称强迫症

```
❌ 每个小节都是"现象 → 分析 → 建议"，重复 4 次
❌ 每段都差不多长（300 字 / 300 字 / 300 字）
❌ 每个论点都配一个例子、一个数据、一句金句（机械排列）

✅ 段落长短要有节奏变化（长段 + 短段 + 一句话段）
✅ 论证方式要有变化（这段用故事，下段用数据，再下段用类比）
✅ 读者的注意力是波浪形的，文章结构也应该是
```

### 5.2 平均用力

```
❌ 5 个论点每个写 400 字，平均分配
❌ 每个论点的重要性看起来一样

✅ 核心论点花最多篇幅，次要论点可以一句话带过
✅ 如果 5 个论点一样重要 → 你还没想清楚哪个最重要
```

### 5.3 缺乏呼吸感

```
❌ 连续 5 个长段落（每段 200+ 字），没有短句、没有留白
❌ 连续 5 个短句段落，没有展开、没有深度

✅ 长段落之后跟一个短句段落（让读者喘口气）
✅ 密集信息之后跟一个故事或场景（让读者消化）
✅ 一篇 2000 字的文章，至少应该有 2-3 个"一句话段落"
```

## 六、平台特定反模式

### 6.1 小红书

```
❌ 通篇没有 emoji（在小红书显得冷漠）
❌ emoji 超过 8 个（视觉噪音）
❌ 每段开头都是 emoji（机械感）
❌ 写成公众号的深度长文风格
❌ 用"笔者""本文"等书面语

✅ 像跟闺蜜/朋友说话的语气
✅ emoji 用在关键转折处，不是装饰
```

### 6.2 公众号

```
❌ 开头就是"大家好，我是 XX"（读者不在乎）
❌ 文末"点赞 + 在看 + 转发"三连求（在正文里求，不是文末固定模板里求）
❌ 段落太长没有配图/引用块分隔（手机阅读体验差）

✅ 每 500-800 字考虑一次视觉分隔（引用块、加粗句、短句段）
```

### 6.3 知乎

```
❌ 开头"谢邀"（除非真的被邀请了）
❌ 通篇主观感受没有论据
❌ 写成小红书的感性风格

✅ 知乎读者期待"被说服"，需要逻辑链 + 证据
✅ 可以有态度，但态度需要论证支撑
```

### 6.4 Twitter/X

```
❌ 单条推文超过 200 字还不换行
❌ Thread 每条都以"接上条"开头
❌ 第一条不是钩子（第一条决定了有没有人往下看）

✅ 每条推文独立成立
✅ Thread 第一条 = 最强的钩子
✅ Thread 最后一条 = 最强的金句
```

---

## 使用方式

1. **Step 12 生成初稿时**：每写完一个段落，快速过一遍本文件的检测规则
2. **篇幅适配检查时**：作为最终质检的一部分
3. **金句/钩子生成时**：检查是否触碰了表达反模式
4. **如果不确定是否触犯**：默认"触犯了"，重写比发布烂内容代价小
