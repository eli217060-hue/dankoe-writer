# Dan Koe Writer Skill V4.1

> 研究驱动的内容积木系统，学习 Dan Koe 方法论，创作有深度的爆款内容
>
> **核心理念**：先研究再写作，积木全量生成、动态组装。
> 本 Skill 负责结构与内容，Chinese Style Optimizer 负责文笔优化。

---

## 创作流程

```
Phase 1：理解需求
│  Step 1：用户提出主题
│  Step 2：确认身份 / 平台 / 篇幅
│
Phase 2：研究素材
│  Step 3：联网研究（3-6 次搜索）
│
Phase 3：选择方向
│  Step 4：提供 6+ 切入角度（每个角度关联真实数据）
│  Step 5：用户选择角度
│
Phase 4：深度分析
│  Step 6：AI 内部深度追问（用户不可见）
│  Step 7：向用户询问细节（可答可不答）
│
Phase 5：生成积木
│  Step 8：生成完整积木库（8 类，约 32 块）
│  Step 9：强制多样性检查
│
Phase 6：组装输出
│  Step 10：动态匹配，推荐 3 个组装方案（同平台同篇幅，不同 SOP/调性）
│  Step 11：用户选择方案
│  Step 12：生成初稿
│  Step 13：提供 2-3 个标题选项
│
Phase 7：文笔优化
│  Step 14：传递给 Chinese Style Optimizer
│
Phase 8：完成
│  输出最终版本
│  用户可说"再出一个 XX 平台版本" → 回到 Step 10 用同一批积木重新组装
```

## 各步骤详细规则

### Step 2：确认基础信息

通过 `ask_user_input` 一次性收集：
- 平台（小红书 / 公众号 / 知乎 / Twitter / Newsletter / 其他）
- 篇幅（短文 500 字以内 / 中文 1500-3000 字 / 长文 3000+ 字）
- 作者身份（一句话描述，如"芳疗师""独立开发者"）
- 可选：立场偏好 / 个人经历 / 目标读者 / 语气偏好

### Step 3：联网研究

**必须使用 `web_search` 和 `web_fetch`**，不可跳过。

搜索策略（3-6 次）：
1. 核心主题搜索（了解基本面）
2. 反面 / 争议观点搜索（找到非主流但有道理的角度）
3. 数据 / 研究搜索（找科学依据、统计数据）
4. 跨领域联系搜索（寻找意外关联，这是 Dan Koe 风格的灵魂）
5-6. 根据前面结果做针对性深挖

输出：编译成内部「原始素材摘要」，包含关键数据、研究结论、争议观点。
**用户不可见此摘要，但后续所有步骤都基于它。**

### Step 4：提供切入角度

- 至少 6 个角度
- 每个角度必须关联至少 1 条研究结果或真实数据（不可凭空编造）
- 只呈现：角度名称 + 核心差异 + 为什么值得写
- **不标注 SOP 类型**（SOP 匹配在 Step 10 才发生）

### Step 6：AI 内部深度追问

用户选择角度后，AI 在内部完成以下分析（用户看不到过程）：

```
6a. 这个角度的底层逻辑是什么？（机制层面）
6b. 研究素材中哪些方法/观点支持这个角度？它们的共同点和差异？
6c. 这个角度在哪些特定场景下最有效？哪些场景下会失效？
6d. 有没有跨领域类比可以强化这个角度？
6e. 关键词过载检测：
    → 如果整个分析可以用 1 个词概括，说明追问不够深
    → 继续追问直到至少 2-3 个不同维度的支撑逻辑
```

### Step 7：向用户询问细节

通过 `ask_user_input` 或直接提问，收集个人素材：
- "你有没有相关的个人经历或客户案例？"
- "你的读者最常见的误区是什么？"
- "有没有想特别强调的行动建议？"

**用户可以全部跳过。** 跳过则 AI 用内部分析结果继续。
如果用户补充了细节 → 优先融入故事积木和行动步骤积木。

### Step 8：生成完整积木库

**永远全量生成**，不因篇幅删减。这是素材池，组装时才做筛选。

| 积木类型 | 数量 | 说明 |
|----------|------|------|
| 核心洞察 | ×3 | 深度观点，文章灵魂 |
| 金句 | ×5 | 短小精炼，传播利器 |
| 故事 | ×3 | 标注来源：个人经历 / 研究案例 / 读者场景 |
| 数据 | ×3 | 必须有真实来源，不可编造 |
| 类比 | ×4 | 跨领域比喻，降低理解门槛 |
| 反常识钩子 | ×4 | 打破认知，开场引爆 |
| 行动步骤 | ×5 | 具体方法，立即可做 |
| 问题 | ×3 | 引发思考，互动引导 |

积木内容来源：
- 研究素材（Step 3）提供事实基础
- 深度追问（Step 6）提供分析深度
- 用户细节（Step 7）提供个人素材
- Dan Koe 文章分析（knowledge-base/articles/）提供写作模式参考

### Step 9：强制多样性检查

生成积木后自动执行，**任何一项不通过则重新生成该类积木**：

```
□ 核心洞察（3 条）：是否探索了不同侧面？
  测试：如果 3 条可以用同一句话概括 → 不通过

□ 金句（5 条）：语气是否有变化？
  要求：至少覆盖 3 种语气（犀利 / 温暖 / 哲学 / 幽默 / 挑衅）

□ 类比（4 条）：是否来自 3 个以上不同领域？
  反例：全是科技比喻（CPU / 电脑 / 手机）→ 不通过

□ 故事（3 条）：是否包含不同类型？
  要求：个人经历 / 研究案例 / 读者场景 至少覆盖 2 种

□ 反常识钩子（4 条）：强度是否有梯度？
  要求：轻微好奇 → 中度挑衅 → 强烈颠覆，不能全是同一强度

□ 行动步骤（5 条）：是否覆盖不同类型？
  要求：立即可做 / 应该避免 / 深度练习 至少覆盖 2 种
```

**复读机检测**：
```
提取所有积木的核心关键词，
如果超过 60% 在表达同一个意思 → 判定为"复读机模式"
→ 回到 Step 6 重新做深度追问，寻找更多维度
```

### Step 10：动态匹配组装方案

**原则：积木全量保留，组装时根据平台/篇幅动态挑选。**

#### 10a. 根据篇幅确定组装规则

**短文（500 字以内）**：
- 从完整库中挑选 3-4 块最强积木
- 只有 1 条叙事线（不允许"以前 vs 现在"双线对比）
- 类比可以直接当论证，不需要额外解释段
- 故事最多 1 个且不超过 3 句话
- 行动步骤最多 1 个且不超过 3 步
- 段落之间不需要过渡句（短文节奏是跳跃的）
- 结尾要么是金句要么是行动步骤，不能两个都有
- 不允许出现"总结"段落
- 总段落数不超过 5-6 段
- **参考 `patterns/short-form.md` 的短内容模式**

**中文（1500-3000 字）**：
- 使用 5-7 块积木
- 可以有完整故事展开
- 可以有多步方法论
- 允许 1 次"对比转变"结构

**长文（3000 字以上）**：
- 积木使用不限
- 可以多线叙事
- 完整展开故事、数据、方法论

#### 10b. 从 SOP 库中匹配结构

根据当前主题 + 平台 + 篇幅，从 `sop/content-sop.md` 中挑选 3 个最适合的 SOP 类型。

参考 `patterns/structures.md` 中的结构模式。
参考 `articles/` 中相似主题的文章结构。

#### 10c. 推荐 3 个方案

每个方案展示：
```
方案 N：[名称]
SOP 类型：[反共识型 / 方法论型 / 故事先行型 / ...]
使用积木：[列出编号]
情绪弧线：[一句话描述]
为什么推荐：[简要说明]
```

**方案多样性硬规则**：
- 3 个方案必须使用不同的 SOP 类型
- 情绪弧线必须明显不同（读出来像不同人写的）
- 不是简单地调换积木顺序

### Step 12：生成初稿

- 严格遵守 Step 10a 的篇幅组装规则
- 展开积木时自然融入，不机械堆砌
- 参考 `patterns/` 下的钩子、金句、标题模式
- 参考 `methods/dankoe-methods.md` 中的方法论框架
- 用微故事法（Problem → Amplify → Solution）作为最小写作单元——任何段落都可以从一个相关问题开始

**段落级写作乐高（展开积木时的填充工具）**：

积木系统告诉你"这篇文章由哪些大块组成"。
写作乐高告诉你"这个大块内部怎么展开"。

当展开某块积木、不知道下一句写什么时，循环检查以下选项：

```
□ 痛点 — 用一个相关痛点起头，思路会自然流动
□ 例子 — 扔一个具体例子让观点落地
□ 个人故事 — 3 句话的场景闪回
□ 统计数据 — 找一个真实数字增加权威
□ 隐喻 — 像对小孩解释一样
□ 引用 — 用名言强化观点
□ 重构视角 — 换个角度再看一次
□ 追问为什么 — 问自己 what/how/why，继续深挖
```

**乐高使用规则**：
- 不需要全用，一个段落用 1-2 种就够
- 写到卡住时从上到下过一遍清单，选最自然的
- 短文（500字以内）最多用 2-3 种乐高

### 篇幅适配检查（初稿完成后自动执行）

```
短文检查：
□ 总字数是否在目标范围内？
□ 叙事线是否只有 1 条？
□ 是否出现了"以前 vs 现在"对比段？（不允许）
□ 行动步骤是否超过 3 步？（不允许）
□ 是否有不必要的过渡句 / 总结句？
□ 同一关键词出现是否超过 5 次？（复读机检测）
□ emoji 是否超过 5 个？（小红书可放宽到 8 个）
```
