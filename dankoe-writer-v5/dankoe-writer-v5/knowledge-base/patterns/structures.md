# 结构模式

> 从 Dan Koe 文章中提取的文章结构，与 SOP 库配合使用

---

## 一、层层深入型

```
表面现象 → 原因分析 → 心理层面 → 解决方案 → 行动召唤
```

**Dan Koe 用法**：不在第一层停下来。大多数作者写到"原因分析"就给方法论了，Dan Koe 会多挖一层到心理机制，让读者觉得"别人只告诉我 what，他告诉我 why behind why"。

**示例文章**：The Unpleasant Truth: AI Is Coming For Your Job
- 表面：AI 要取代你的工作
- 原因：平庸工作可以被自动化
- 心理：你的独特性是什么？
- 方案：构建不可替代的技能组合
- 行动：4 个具体步骤

**适合篇幅**：中文、长文

---

## 二、规则列表型

```
Rule 1: [核心主张] → [解释]
Rule 2: [核心主张] → [解释]
...
```

**Dan Koe 用法**：规则之间有递进关系，不是并列罗列。前面的规则是后面的基础，最后的规则是升华。

**示例文章**：12 rules to change your life in 12 months
- 前 4 条：基础认知（目标、身份）
- 中间 4 条：行为改变（习惯、专注）
- 后 4 条：系统构建（创作、复利）

**适合篇幅**：中文、长文

---

## 三、对比论证型

```
[主流观点] → [为什么失效] → [新范式] → [具体方法]
```

**Dan Koe 用法**：先承认旧观点有道理（"它曾经有效"），再解释为什么现在不行了，最后提供替代方案。不是简单地"你错了我对了"。

**示例文章**：The death of value-based content
- 旧：价值型内容有效
- 为什么失效：每个人都在做，AI 也会做
- 新范式：视角型内容
- 方法：如何构建独特视角

**适合篇幅**：中文、长文

---

## 四、问题→分析→方案型

```
[痛点问题] → [深度分析为什么] → [解决框架] → [行动步骤]
```

**Dan Koe 用法**：分析部分占文章最大比重（40-50%），因为 Dan Koe 认为"理解问题比解决问题更重要"。

**示例文章**：How to fix your entire life in 1 day
- 痛点：人生感觉一团糟
- 分析：目标镜头理论 + 身份-行为模型
- 框架：一天重构法
- 步骤：具体的日程安排

**适合篇幅**：中文、长文

---

## 五、故事先行型

```
[个人故事/场景] → [从故事中提炼洞察] → [扩展到普遍原理] → [读者可以做什么]
```

**Dan Koe 用法**：故事不是开胃菜，而是证据。故事里已经包含了核心论点的种子。

**适合篇幅**：所有篇幅（短文用 3 句话版本，长文用完整叙事）

---

## 短文专用结构

短文不使用上述完整结构，而是用以下简化版：

### 结构 A：钩子 + 洞察
```
[反常识钩子] → [核心洞察，1-2 句话展开] → [结尾留白]
```

### 结构 B：故事 + 点睛
```
[3 句话场景] → [类比或金句点睛]
```

### 结构 C：洞察 + 行动
```
[核心洞察] → [1 个立即可做的行动] → [开放式问题收尾]
```

**参见 `short-form.md` 获取更详细的短内容规则、节奏示范和策略性模糊规则。**

## APAG 长文框架（V5 核心）

这是 Dan Koe 长文的核心结构。V5 中，所有长文（1500 字以上）默认使用此框架。

```
A = Attention（吸引注意力）
  - 开头 200-500 字
  - "攻击敌人"：说明大多数人做错了什么
  - 可用：个人故事 / 直接陈述 / 惊人数据
  - 目标：抓住注意力 + 筛选读者 + 打开好奇心缺口

P = Perspective（构建视角）
  - 描绘读者当前的错误视角及其带来的痛苦
  - 放大问题，让读者觉得"这比我想的严重"
  - 可用：问题、收益、案例、隐喻、名言、故事、对比
  - 目标：让读者感到被理解

A = Advantage（凸显优势）
  - 展示你的视角为什么更好
  - 提供新概念、新框架、新思路
  - 可用：研究数据、跨领域类比、个人经验
  - 目标：制造"啊哈时刻"

G = Gamify（游戏化落地）
  - 给出具体、低门槛的行动步骤
  - 步骤要小到读者"没理由不做"
  - 目标：让读者觉得"我今天就能开始"
```

### APAG 使用注意

- 各部分**没有固定长度**，根据主题灵活调整
- Perspective 可以占文章的一半（如果问题需要深入分析）
- Gamify 可以占文章的 90%（如果是方法论型内容）
- 这是**柔性指导**，不是死板结构
- 与大纲模板中的"3 个核心观点"对应：观点 1 通常落在 Perspective，观点 2 落在 Advantage，观点 3 落在 Gamify

### APAG 情感弧线

```
Attention：制造"我也是这样"的共鸣 或 "等等，什么？"的好奇
Perspective：放大痛苦，"这个问题比我想的严重"
Advantage：提供顿悟，"原来可以这样看"
Gamify：给出行动，"我今天就能开始做"
结尾：留下回响，"这句话我要记住"
```

## 结构与 SOP 的关系（V5 说明）

V5 中，SOP 降级为参考。大纲前置后，结构由大纲决定，不再由 SOP 分类驱动。

| 结构 | 适用场景 |
|------|----------|
| APAG 框架 | 所有长文的默认结构 |
| 层层深入型 | 需要多层分析的深度话题 |
| 对比论证型 | 旧观点 vs 新观点的主题 |
| 规则列表型 | 经验总结型内容 |
| 故事先行型 | 个人经历驱动的内容 |
| 短文专用结构 | 500 字以内的短内容 |
