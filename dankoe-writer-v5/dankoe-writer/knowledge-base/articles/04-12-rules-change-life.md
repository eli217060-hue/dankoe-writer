## 可复用的模式

### "规则型文章"模板

```markdown
# [数字] rules to [效果] in [时间]

[副标题/承诺]

## Rule 1: [第一条]
[内容]

## Rule 2: [第二条]
[内容]

...

## Rule [N]: [最后一条]
[最深刻的洞察]

---

If you only take one thing from this...
[核心信息浓缩]
```

### 规则设计原则

1. **独立性**: 每条规则可单独理解和执行
2. **递进性**: 规则之间有逻辑关系
3. **完整性**: 覆盖主题的所有关键方面
4. **可执行**: 每条规则都有行动暗示

## 核心主题贯穿

### 身份-行为-结果模型

```
身份 (你是谁)
  ↓
行为 (你做什么)
  ↓
结果 (你得到什么)
```

**关键洞察**: 大多数人试图跳过身份，直接改变行为。

### 迭代能力作为核心竞争力

```
问题 → 实验 → 反馈 → 调整 → 重复
```

低智力 = 在问题上卡住
高智力 = 持续迭代直到解决
