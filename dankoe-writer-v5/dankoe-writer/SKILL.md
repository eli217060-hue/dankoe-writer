# Dan Koe Writer Skill V5

> 研究驱动的内容积木系统，学习 Dan Koe 方法论，创作有深度的爆款内容
>
> **核心理念**：先研究再写作，大纲前置、积木服务于大纲。
> 本 Skill 负责结构与内容，Chinese Style Optimizer 负责文笔优化。

---

## 创作流程

```
Phase 1：理解需求
│  Step 1：用户提出主题
│  Step 2：确认身份 / 平台 / 篇幅

Phase 2：研究素材
│  Step 3：联网研究（3-6 次搜索）

Phase 3：生成大纲（核心）
│  Step 4：AI 生成 5 个完整大纲
│           每个大纲包含：核心悖论 + 敌人/错误观点 + 转变弧线
│           + 核心观点 + 读者异议 + 难忘结尾
│  Step 5：用户选择 1 个大纲 + 补充个人细节（可选）

Phase 4：定向填充
│  Step 6：为大纲每个位置生成定向积木（带情感功能标签）
│  Step 7：积木质量检查 + 多样性检查

Phase 5：写作
│  Step 8：推荐 3 种情感原型的写法
│  Step 9：用户选择情感原型
│  Step 10：生成初稿
│  Step 11：篇幅适配检查
│  Step 12：提供 2-3 个标题选项

Phase 6：文笔优化
│  Step 13：传递给 Chinese Style Optimizer

Phase 7：完成
│  输出最终版本
│  主动推荐复用方向："这批素材还可以写 XX 平台版本"
│  用户可说"再出一个 XX 平台版本" → 回到 Step 8
```

### 用户决策点

| 决策 | 内容 |
|------|------|
| 决策 1 | 从 5 个完整大纲中选 1 个（可同时补充个人细节） |
| 决策 2 | 从 3 种情感原型中选 1 种 |

用户只需做 2 次选择，每次决策信息量充足。

## 各步骤详细规则

### Step 2：确认基础信息

通过 `ask_user_input` 一次性收集：
- 平台（小红书 / 公众号 / 知乎 / Twitter / Newsletter / 其他）
- 篇幅（短文 500 字以内 / 中文 1500-3000 字 / 长文 3000+ 字）
- 作者身份（一句话描述，如"芳疗师""独立开发者"）
- 可选：立场偏好 / 个人经历 / 目标读者 / 语气偏好

### Step 3：联网研究

**必须使用 `web_search` 和 `web_fetch`**，不可跳过。

搜索策略（3-6 次）：
1. 核心主题搜索（了解基本面）
2. 反面 / 争议观点搜索（找到非主流但有道理的角度）
3. 数据 / 研究搜索（找科学依据、统计数据）
4. 跨领域联系搜索（寻找意外关联，这是 Dan Koe 风格的灵魂）
5-6. 根据前面结果做针对性深挖

输出：编译成内部「原始素材摘要」，包含关键数据、研究结论、争议观点。
**用户不可见此摘要，但后续所有步骤都基于它。**

### Step 4：生成 5 个完整大纲

AI 在生成大纲前，**内部**必须先完成以下分析（用户不可见）：

```
4a. 这个主题的底层逻辑是什么？（机制层面）
4b. 研究素材中哪些方法/观点支持哪些角度？共同点和差异？
4c. 各角度在哪些场景下最有效？哪些场景下会失效？
4d. 有没有跨领域类比可以强化？
4e. 关键词过载检测：
    → 如果分析可以用 1 个词概括，说明追问不够深
    → 继续追问直到至少 2-3 个不同维度的支撑逻辑
4f. 读者在这个话题上，已经感受到但说不出的是什么？
    → 这是核心悖论的来源
    → "改变人们的帖子不是告诉他们不知道的东西，
       而是表达他们感受到但无法表达的东西"
4g. 读者最可能的反驳是什么？
    → 这是读者异议的来源
    → 提前回应异议能防止读者中途退出
4h. 如果用一句话改变读者的信念，那句话是什么？
    → 这是难忘结尾的来源
```

然后根据分析结果，生成 **5 个完整大纲**。每个大纲根据篇幅使用不同模板：

#### 短文大纲模板（小红书/推文，500 字以内）

```
大纲 N：[角度名称]

核心悖论：[一句话，抓住注意力的反常真理]
  表述 1：[最短版本]
  表述 2：[换一种说法]
  表述 3：[最尖锐版本]

敌人/错误观点：[大多数人做错了什么，1-2 句话]

转变弧线：[从旧认知 → 新认知，一句话描述]

关键示例：[1-2 个具体、短小的例子]

读者异议：[读者最可能的反驳，1-2 个]
  异议 1：[...]
  回应方向：[不需要直接反驳，暗示即可]

难忘结尾：[一句话，能让人截图转发的金句]

情绪走向：[这篇文章让读者从 __ 感受到 __]
```

#### 长文大纲模板（公众号/Newsletter，1500 字以上）

```
大纲 N：[角度名称]

核心悖论：[一句话]
  表述 1-3：[同上]

敌人/错误观点：[大多数人的做法或认知，2-3 句话]

转变弧线：挑战 → 揭示 → 超越
  挑战：[读者当前面临的问题]
  揭示：[打破旧认知的关键洞察]
  超越：[新的可能性]

3 个核心观点（对应 APAG 中的 Perspective → Advantage → Gamify）：
  观点 1：[标题] — 解决什么问题 — 需要什么素材（故事？数据？类比？）
  观点 2：[标题] — 提供什么洞察 — 需要什么素材
  观点 3：[标题] — 给出什么行动 — 需要什么素材

读者异议：[2-3 个]
  异议 1：[...]
  异议 2：[...]

理想陈述：[读完后读者应该成为什么样的人 / 相信什么]

难忘结尾：[一句话]

情绪走向：[从 __ 到 __ 到 __]
```

### Step 5：用户选择大纲

通过 `ask_user_input` 让用户选择 1 个大纲。
同时询问："你有没有相关的个人经历或客户案例？读者最常见的误区是什么？"
**用户可以全部跳过。**

### Step 6：定向填充积木

**不再是"8 类 32 块"通用积木，而是"大纲需要什么就生成什么"。**

根据大纲中每个位置的需求，生成定向积木。每块积木必须标注**情感功能标签**：

```
积木标签格式：[类型] | [情感功能] | [内容]

情感功能分类：
- 颠覆认知：让读者停下来重新思考
- 制造共鸣：让读者觉得"他在说我"
- 秒懂：降低理解门槛
- 建立信任：用数据/权威增加可信度
- 让人截图：可传播的金句
- 制造紧迫：让读者觉得"我必须改变"
- 提供希望：让读者觉得"我也能做到"

示例：
核心洞察 | 颠覆认知 | "薰衣草只对一种失眠有效，但失眠有三种"
金句 | 让人截图 | "找对了油，身体自己就会记住该睡了"
故事 | 制造共鸣 | "我用薰衣草用了半年，时灵时不灵..."
类比 | 秒懂 | "用薰衣草治所有失眠，就像用感冒药治所有发烧"
数据 | 建立信任 | "芳樟醇能促进 5-HT 分泌（芳疗研究综述）"
```

积木内容来源：
- 研究素材（Step 3）提供事实基础
- 内部分析（Step 4）提供分析深度
- 用户细节（Step 5）提供个人素材
- Dan Koe 文章分析（knowledge-base/articles/）提供写作模式参考

### Step 7：积木质量检查 + 多样性检查

#### 质量检查（每块积木必须通过）

```
□ 画面感测试：能让读者在脑海中看到一个场景吗？
  ✗ "她很焦虑"
  ✓ "凌晨三点，她盯着天花板"

□ 12 岁测试：12 岁的孩子能听懂吗？
  如果不能，需要配一个类比或例子

□ "所以呢"测试：读者读完会说"所以呢"吗？
  如果会，这块积木不够有力，需要连接到读者的生活

□ 截图测试（仅限金句）：读者会截图发朋友圈吗？
  如果不会，金句不够好

□ 真实性测试（仅限数据）：有可验证的来源吗？
  不可编造数据
```

#### 多样性检查

```
□ 核心洞察：是否探索了不同侧面？
  测试：如果可以用同一句话概括 → 不通过

□ 金句：语气是否有变化？
  要求：至少覆盖 3 种语气（犀利 / 温暖 / 哲学 / 幽默 / 挑衅）

□ 类比：是否来自 3 个以上不同领域？
  反例：全是科技比喻 → 不通过

□ 故事：是否包含不同类型？
  要求：个人经历 / 研究案例 / 读者场景 至少覆盖 2 种
```

**复读机检测**：
```
提取所有积木的核心关键词，
如果超过 60% 在表达同一个意思 → 判定为"复读机模式"
→ 回到 Step 4 内部分析，寻找更多维度
```

### Step 8：推荐 3 种情感原型

基于同一个大纲，推荐 3 种不同情感原型的写法（参见 `methods/dankoe-methods.md` 中的情感原型详解）：

```
写法 1：耐心观察者
  情绪弧线：挣扎 → 验证 → 希望
  语气：温暖、理解、"我懂你"

写法 2：戏剧性预言家
  情绪弧线：危机 → 破坏 → 重生
  语气：强烈、命令式、"你必须改变"

写法 3：静默颠覆者
  情绪弧线：观察 → 对比 → 反思
  语气：冷静、留白、"我经常想这个"
```

每种写法展示：情绪弧线 + 开头示意 + 结尾示意 + 为什么推荐

### Step 10：生成初稿

- 严格遵守篇幅组装规则（见下方）
- 遵守情感弧线公式（见下方）
- 展开积木时自然融入，不机械堆砌
- 参考 `patterns/` 下的钩子、金句、标题模式
- 参考 `methods/dankoe-methods.md` 中的方法论框架
- 长文使用 APAG 框架（参见 `patterns/structures.md`）

## 情感弧线公式

每篇文章必须有明确的情感弧线，不只是信息结构。

### 短文情感弧线

```
- 读者在第 1 段应该感受到：[被说中 / 被挑衅 / 好奇]
- 读者在中间应该感受到：[被理解 / 被颠覆 / 恍然大悟]
- 读者在结尾应该感受到：[被激励 / 被触动 / 想转发]
```

### 长文情感弧线

```
- 开头（钩子）：制造"我也是这样"的共鸣 或 "等等，什么？"的好奇
- Perspective（视角）：放大痛苦，让读者觉得"这个问题比我想的严重"
- Advantage（优势）：提供顿悟，"原来可以这样看"
- Gamify（游戏化）：给出行动，"我今天就能开始做"
- 结尾：留下回响，"这句话我要记住"
```

## 篇幅组装规则

### 短文（500 字以内）

- 从积木中挑选 3-4 块最强积木
- 只有 1 条叙事线（不允许"以前 vs 现在"双线对比）
- 类比可以直接当论证，不需要额外解释段
- 故事最多 1 个且不超过 3 句话
- 行动步骤最多 1 个且不超过 3 步
- 段落之间不需要过渡句（短文节奏是跳跃的）
- 结尾要么是金句要么是行动步骤，不能两个都有
- 不允许出现"总结"段落
- 总段落数不超过 5-6 段
- **参考 `patterns/short-form.md` 的短内容模式**

**短文"保留"规则**：
```
□ 至少有一个未完全回答的问题（让读者自己想）
□ 结尾不总结，用开放式表达或金句收尾
□ 不解释每一个观点——描述，让读者自己得出结论
□ 使用策略性模糊：不要把所有东西都说得太具体
  例如："有意义的东西""那个改变一切的时刻"
  让读者把自己的处境投射进去

永远不要在短文中：
□ 给出完整的解决方案清单
□ 用"总之""综上"收尾
□ 解释每一个类比（类比即论证）
□ 告诉读者"接下来我要说的是..."
```

### 中文（1500-3000 字）

- 使用 5-7 块积木
- 可以有完整故事展开
- 可以有多步方法论
- 允许 1 次"对比转变"结构

### 长文（3000 字以上）

- 积木使用不限
- 可以多线叙事
- 完整展开故事、数据、方法论
- **核心结构使用 APAG 框架**（参见 `patterns/structures.md`）

### 篇幅适配检查（初稿完成后自动执行）

```
短文检查：
□ 总字数是否在目标范围内？
□ 叙事线是否只有 1 条？
□ 是否出现了"以前 vs 现在"对比段？（不允许）
□ 行动步骤是否超过 3 步？（不允许）
□ 是否有不必要的过渡句 / 总结句？
□ 同一关键词出现是否超过 5 次？（复读机检测）
□ emoji 是否超过 5 个？（小红书可放宽到 8 个）
□ 是否遵守了"保留"规则？
□ 开头是否是"攻击敌人"而非平铺直叙？
□ 情感弧线是否清晰？
```
