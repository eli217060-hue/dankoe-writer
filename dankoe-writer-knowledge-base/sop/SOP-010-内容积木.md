# SOP-010: 内容积木处理流程

> 内容积木 = 核心观点 + 联系 + 场景

---

## 一、内容积木定义

### 什么是内容积木？

内容积木是文章的最小构建单位，每个积木包含三个维度：

```
内容积木 = 核心观点 + 联系 + 场景

核心观点：一个有洞察的论点（不是金句，是完整的观点）
联系：
• 与哪个观点是因果关系？
• 与哪个观点是对比关系？
• 与哪个观点是递进关系？
场景：
• 适合放在文章哪个位置？
• 起什么作用？
```

### 内容积木 ≠ 金句

| 金句 | 内容积木 |
|------|----------|
| 一句话的精炼表达 | 完整的观点单元 |
| 独立存在 | 有明确的联系和位置 |
| 追求"好记" | 追求"好用" |
| 装饰性 | 功能性 |

## 二、积木类型与功能

### 钩子型积木

**功能**：抓住注意力，制造共鸣

**特征**：
- 触发"我就是这样的"情绪
- 提出一个反直觉的问题
- 揭示一个隐藏的矛盾

**示例**：
```
核心观点：焦虑来临的时刻，恰恰是身体在请求一个"暂停键"
联系：与下一块积木是因果关系
场景：开头，制造共鸣
```

### 论证型积木

**功能**：展开论述，提供证据

**特征**：
- 解释"为什么"
- 提供具体例子
- 建立逻辑链条

**示例**：
```
核心观点：精油不治病，但它能建立一个"呼吸间"——你和自己相处的3分钟
联系：承接上一块，引出下一块
场景：中段，展开论述
```

### 转折型积木

**功能**：转换视角，升华主题

**特征**：
- "但真正重要的是..."
- "表面上看...实际上..."
- 从现象到本质

**示例**：
```
核心观点：最有效的不是精油本身，而是你给自己创造的那个"仪式时刻"
联系：与上一块是递进关系
场景：转折升华
```

### 收尾型积木

**功能**：总结核心，留下余味

**特征**：
- 回应开头
- 提炼核心洞察
- 给出行动暗示

**示例**：
```
核心观点：焦虑没有消失，但它不再是敌人，变成了"该和自己待一会儿"的信号
联系：呼应开头，完成闭环
场景：结尾，留下余味
```

## 三、积木生成流程

### 流程图

```
用户选择角度
     ↓
① 深度追问
  - 这个角度的底层规律是什么？
  - 与常识有什么不同？
  - 能解决什么真实问题？
     ↓
② 确认细节（AI 自行完成，不需用户输入）
  - 目标读者的具体痛点
  - 可用的素材和证据
  - 情绪流动的路径
     ↓
③ 生成内容积木
  - 每个积木：核心观点 + 联系 + 场景
  - 积木之间有明确的逻辑关系
  - 覆盖开头、中段、转折、结尾
     ↓
④ 匹配结构
  - 观点确定后才选择结构
  - 结构服务于观点表达
  - 不是先有结构再填内容
     ↓
⑤ 生成初稿
  - 按积木顺序组装
  - 保持积木之间的流畅过渡
  - 控制整体篇幅
```

### 关键原则

1. **观点优先，结构跟随**
   - 先有观点，再选结构
   - 结构服务于表达
   - 不是填空题

2. **积木独立又相连**
   - 每个积木可独立理解
   - 积木之间有明确联系
   - 形成逻辑链条

3. **场景决定位置**
   - 每个积木有明确的"位置标签"
   - 位置决定写作方式
   - 同样的观点在不同位置写法不同

## 四、积木模板

### 模板 A：钩子-展开-升华（短内容）

```
积木A（钩子型）
- 核心观点：[反直觉开场]
- 联系：引出积木B
- 场景：开头

积木B（论证型）
- 核心观点：[展开论述]
- 联系：承接A，引出C
- 场景：中段

积木C（转折型）
- 核心观点：[升华洞察]
- 联系：与B递进
- 场景：转折升华
```

### 模板 B：问题-分析-方案（方法论）

```
积木A（钩子型）
- 核心观点：[揭示问题]
- 联系：引出分析
- 场景：开头

积木B（论证型）
- 核心观点：[深度分析原因]
- 联系：因果关系
- 场景：分析段

积木C（论证型）
- 核心观点：[提供解决方案]
- 联系：承接B的问题
- 场景：方案段

积木D（收尾型）
- 核心观点：[行动召唤]
- 联系：呼应A
- 场景：结尾
```

### 模板 C：对比-选择-理由（决策型）

```
积木A（钩子型）
- 核心观点：[呈现两难选择]
- 联系：引出对比
- 场景：开头

积木B（论证型）
- 核心观点：[选项1分析]
- 联系：对比关系
- 场景：第一选项

积木C（论证型）
- 核心观点：[选项2分析]
- 联系：对比关系
- 场景：第二选项

积木D（转折型）
- 核心观点：[推荐选择及理由]
- 联系：综合B和C
- 场景：结论
```

## 五、示例演示

### 输入
- 主题：焦虑
- 身份：精油爱好者
- 平台：小红书
- 篇幅：短
- 选择角度：焦虑不是敌人，而是身体在提醒你需要一个"仪式感"

### 积木生成

```
积木A（钩子型）
- 核心观点：焦虑来临的时刻，恰恰是身体在请求一个"暂停键"
- 联系：与积木B是因果关系
- 场景：开头，制造共鸣

积木B（论证型）
- 核心观点：精油不治病，但它能建立一个"呼吸间"——你和自己相处的3分钟
- 联系：承接积木A，引出积木C
- 场景：中段，展开论述

积木C（转折型）
- 核心观点：最有效的不是精油本身，而是你给自己创造的那个"仪式时刻"
- 联系：与积木B是递进关系
- 场景：转折升华
```

### 结构匹配

选用 [钩子-展开-升华] 结构，适合小红书短内容的情绪流动

### 初稿输出

---

焦虑的时候，我以前会刷手机。

越刷越慌。

后来我学会了给自己一个"暂停键"。

不是什么大道理。

就是在桌角放一瓶薰衣草精油。

焦虑来了——打开盖子，深呼吸三下。

就三下。

**精油不治病。**

**但它能给你建立一个"呼吸间"——**

**一个只属于你和自己的三分钟。**

那些原本要被焦虑吞噬的时间，

变成了你主动选择的"仪式时刻"。

焦虑没有消失。

但它不再是你的敌人。

它变成了一个信号：

**"嘿，该停下来，和自己待一会儿了。"**
