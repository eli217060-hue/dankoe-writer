# SOP-010: 内容积木系统

> 从 Dan Koe 文章中学习，动态生成积木，灵活组装成文

---

## 一、核心设计原则

### 积木生成：不是模板填空，是学以致用

```
Dan Koe 文章预分析
     ↓
提取模式（心理机制、论证逻辑、类比方法...）
     ↓
应用到当前主题
     ↓
动态生成积木
```

**关键点**：
- 积木类型框架是固定的（8类）
- 具体积木内容是动态生成的
- 生成依据来自对 Dan Koe 文章的分析

### 组装方案：不是写死的，是分析出来的

```
Dan Koe 文章预分析
     ↓
提取结构模式
     - 钩子→展开→升华
     - 问题→分析→方案
     - 对比→选择→理由
     - ...
     ↓
根据当前主题适配
     ↓
生成多个组装方案
```

**关键点**：
- 方案数量不固定（可能是3个，也可能是4-5个）
- 方案内容不固定（基于 Dan 的真实结构）
- 方案要标注：结构来源 + 适用场景 + 调性

---

## 二、积木库架构

### 8类积木框架

```
内容积木库
├── 核心洞察    # 深度观点，文章的灵魂
├── 金句        # 短小精炼，传播利器
├── 故事        # 场景代入，证据支撑
├── 数据        # 科学背书，可信度加成
├── 类比        # 形象比喻，降低理解门槛
├── 反常识钩子  # 开场引爆，打破认知
├── 行动步骤    # 具体方法，立即可做
└── 问题        # 互动引导，评论区杀手
```

### 积木内容来源

从 Dan Koe 文章分析中提取的模式：

| 分析维度 | 提取内容 | 应用方式 |
|----------|----------|----------|
| 心理触发机制 | 恐惧、身份危机、信息差... | 用于生成反常识钩子、核心洞察 |
| 论证逻辑 | 身份-行为-结果、目标镜头理论... | 用于生成核心洞察 |
| 类比方法 | 电脑关机、日落、办公室... | 用于生成类比积木 |
| 数据引用 | 研究数据、百分比、时间... | 用于生成数据积木 |
| 故事结构 | 个人转变、翻车案例、研究案例... | 用于生成故事积木 |
| 行动框架 | 30分钟仪式、分步方法... | 用于生成行动步骤 |

### Dan Koe 常用心理机制

| 心理机制 | Dan 的用法 | 生成积木类型 |
|----------|------------|--------------|
| 恐惧营销 | "AI is coming for your job" | 反常识钩子 |
| 身份危机 | "You aren't who you need to be" | 核心洞察、金句 |
| 反共识 | "The death of value content" | 反常识钩子、核心洞察 |
| 信息差 | "CEO-level insight" | 核心洞察、数据 |
| 目标重构 | "Goal as lens of perception" | 核心洞察、类比 |
| 行为诊断 | "Most people fail because..." | 核心洞察、故事 |

## 三、完整工作流程

```
用户提出主题
     ↓
【Step 1】确认基础框架
     - 身份（你是谁/想成为谁）
     - 平台（小红书/公众号/知乎...）
     - 篇幅（短/中/长）
     ↓
【Step 2】真实搜索 + 提供切入角度
     - AI 使用搜索工具搜索主题相关资料
     - 基于搜索结果提炼洞察
     - 提供 6+ 个切入角度
     - 只呈现：观点 + 差异 + 价值
     - 不标注类型（让用户凭感觉选）
     ↓
【用户决策 1】选择切入角度
     ↓
【Step 3】深度追问
     - AI 分析该角度的底层逻辑
     - 分析不同方法的共同点和差异
     - 找出哪些方法在特定场景下最有效
     - 挖到本质，不停留在表面
     ↓
【Step 4】询问用户细节
     - AI 提问：有没有特定场景/人群/案例想聚焦？
     - AI 提问：有没有想强调的独特观点？
     - AI 提问：其他任何想加入的创意？
     ↓
【用户决策 2】提供细节（可选）
     - 用户回答：提供具体细节/创意
     - 或用户回答：没有，你看着来
     ↓
【Step 5】生成积木库
     - 调用 Dan Koe 文章分析（心理机制、论证逻辑...）
     - 结合用户提供的细节/创意（如有）
     - 动态生成 8 类积木（约 32 个）
     - 核心洞察 x3
     - 金句 x5
     - 故事 x3
     - 数据 x4
     - 类比 x4
     - 反常识钩子 x5
     - 行动步骤 x5
     - 问题 x3
     ↓
【Step 6】提供组装方案
     - 从 Dan Koe 文章结构分析提取
     - 提供 3-5 个方案
     - 标注：结构来源 + 适用场景 + 调性
     ↓
【用户决策 3】选择组装方式
     - 选项 A：采用推荐方案之一
     - 选项 B：自定义混搭（指定用哪些积木）
     ↓
【Step 7】组装输出初稿
     ↓
【用户决策 4】确认初稿
     - 满意 → 继续
     - 需修改 → 指出调整方向
     ↓
【Step 8】提供标题选项
     - 提供 4-5 个标题
     - 不同类型：情绪共鸣型/反差吸引型/场景代入型...
     ↓
【用户决策 5】选择标题
     ↓
完成
```

### 用户决策点汇总

| 步骤 | 用户决策 | 选项数量 | 必选/可选 |
|------|----------|----------|-----------|
| Step 2 | 选择切入角度 | 6+ 个 | 必选 |
| Step 4 | 提供细节/创意 | 自由输入 | 可选 |
| Step 6 | 选择组装方式 | 推荐方案/自定义混搭 | 必选 |
| Step 7 | 确认初稿 | 满意/需修改 | 必选 |
| Step 8 | 选择标题 | 4-5 个 | 必选 |

## 四、关键环节详解

### Step 2：真实搜索

AI 使用搜索工具（如 web_search）搜索主题相关资料：

```
搜索示例：
- 主题 "焦虑" → 搜索 "焦虑的原因 研究"、"焦虑管理方法"
- 主题 "失眠" → 搜索 "失眠 研究"、"精油 助眠 机制"
- 主题 "拖延" → 搜索 "拖延症 心理学"、"拖延与完美主义"

搜索结果用于：
- 提炼有依据的洞察
- 发现不同视角
- 找到支撑数据
```

### Step 3：深度追问

用户选择切入角度后，AI 进行深度分析：

```
追问方向：
1. 底层逻辑：这个角度的本质是什么？
2. 方法对比：不同方法的共同点和差异是什么？
3. 场景适配：哪些方法在特定场景下最有效？
4. 反常识点：有什么与常识相悖的发现？

示例（用户选择"焦虑是信号"角度）：
- 焦虑作为"信号"的本质是什么？→ 身体的预警机制
- 不同应对方法的差异？→ 压制 vs 接纳 vs 利用
- 什么场景下哪种方法有效？→ 短期压力用行动，长期焦虑用认知重构
- 反常识点？→ 焦虑不是敌人，是资源
```

### Step 4：询问用户细节

AI 主动询问，用户可选择回答或不回答：

```
AI 提问模板：
1. "有没有特定场景或人群想聚焦？"
   - 例：职场焦虑？学生焦虑？新手妈妈？
   
2. "有没有想强调的独特观点？"
   - 例：从身体角度切入？从认知角度切入？
   
3. "其他任何想加入的创意？"
   - 例：想用某个案例？想结合某个理论？

用户回答示例：
- "聚焦职场焦虑，尤其是开会前的紧张"
- "想强调焦虑其实是身体在帮我们准备"
- "没有特别想法，你看着来"
```

## 五、组装方案生成

### 方案来源：Dan Koe 文章结构分析

Dan Koe 的文章不是随机写的，有可识别的结构模式：

| 结构模式 | Dan 的使用场景 | 示例文章 |
|----------|----------------|----------|
| 钩子→展开→升华 | 短内容、观点输出 | 多篇 Newsletter |
| 问题→分析→方案 | 方法论分享 | "Fix your life in 1 day" |
| 对比→选择→理由 | 决策建议 | "Multiple interests" |
| 规则→递进→收尾 | 经验总结 | "12 rules to change your life" |
| 死亡→原因→新范式 | 反共识观点 | "Death of value content" |

### 方案展示格式

```
方案 1：[名称]
结构：[具体流程，如：反常识钩子 → 核心洞察 → 类比 → 行动步骤 → 金句]
来源：Dan Koe "[文章名]" 的结构变体
调性：[专业/温柔/犀利...]
适用：[什么场景、什么平台]
```

### 方案数量

- 不固定为3个
- 根据主题适配度生成
- 通常 3-5 个有明显差异的方案

## 六、用户选择环节

### 选项 A：采用推荐方案

```
用户：用方案2
AI：[按方案2组装输出]
```

### 选项 B：自定义混搭

```
用户：我想用反常识钩子#4 + 核心洞察#2 + 故事#1 + 行动步骤#3
AI：[按指定积木组装输出]
```

### 选项 C：修改积木

```
用户：核心洞察#2 再深入一点
AI：[重新生成该积木，再次确认]
```

## 七、完整示例

### 输入
- 主题：失眠
- 身份：芳疗师
- 平台：小红书
- 篅幅：中
- 选择角度：精油是"关机键"，不是安眠药

### Step 1: 调用 Dan Koe 分析

```
心理触发点：反共识、身份重构、行为诊断
论证逻辑：问题本质 → 错误方法 → 正确路径
类比方法：电脑关机、日落信号
```

### Step 2: 生成积木库

**核心洞察**
```
#1：精油不是安眠药，它是"关机键"。失眠的本质是大脑卡在白天模式，精油通过嗅觉神经直达边缘系统，帮大脑完成从兴奋到放松的模式切换。

#2：嗅觉是唯一一个不经过"理性大脑"就能直达情绪中枢的感官。这就是为什么闻到一个味道你会先放松、后才意识到自己放松了——精油绕过了你那个"停不下来的脑子"。

#3：失眠的人最大的误区是"努力睡觉"。越努力越清醒，因为"努力"本身就是一种大脑激活状态。精油的价值不是让你"更努力入睡"，而是让你"不需要努力"。
```

**金句**
```
#1：你的失眠不是因为不够累，是因为大脑不知道怎么停下来。
#2：精油不治失眠，它治的是"你躺下了但大脑还在开会"。
#3：最好的助眠，不是让你昏过去，是让你的身体想起"原来可以松下来"。
#4：安眠药是强制关机，精油是让电脑自己进入休眠。
#5：你缺的不是一瓶精油，你缺的是一个"从白天切换到夜晚"的仪式。
```

**故事**
```
#1 [可创作场景]：凌晨一点，你躺在床上，身体已经累得不行，但脑子里还在复盘今天的会议、明天的 deadline、上周没回的微信……

#2 [来自研究]：日本鹿儿岛大学2018年实验——把焦虑的小鼠暴露在芳樟醇蒸气下，焦虑程度大幅下降。

#3 [来自研究]：上海一位消费者买了薰衣草精油想助眠，用了之后反而更兴奋。后来发现那瓶精油掺了合成香精。
```

**数据**
```
#1：吸入薰衣草精油后，大脑产生更多θ波和α波（放松状态的脑电波）。
#2：国内研究证实1%浓度的薰衣草精油对原发性失眠有效。
#3：《中国睡眠研究报告2024》数据，国人平均睡眠指数仅62.61分。
#4：芳香按摩比单纯嗅吸效果更明显（56人随机对照实验）。
```

**类比**
```
#1：精油是"人造日落"。自然界里动物靠日落切换到休息模式，但现代人的手机屏幕把日落取消了。
#2：安眠药是强制关机（长按电源键），精油是让系统自己进入休眠模式。
#3：你的大脑像一间灯火通明的办公室，精油是一盏一盏帮你关灯。
#4：嗅觉是大脑的"后门"，绕过你那个停不下来的思考系统。
```

**反常识钩子**
```
#1：做了X年芳疗师，我从不跟客户说"这个精油能治失眠"。
#2：用了精油还是睡不着？不是精油没用，是你用的时机全错了。
#3：失眠最大的敌人不是压力，是你"努力想睡着"这件事本身。
#4：薰衣草是最有名的助眠精油——但你闻到的那瓶，可能根本不是真的。
#5：我建议所有失眠的客户：别急着"治失眠"，先学会"关机"。
```

**行动步骤**
```
#1 [30分钟仪式]：睡前30分钟开香薰机，滴3滴真正薰衣草（Lavandula angustifolia）。
#2 [辨真假]：买精油看瓶身有没有标拉丁学名。
#3 [按摩加持]：试试在手腕内侧和太阳穴各涂一滴（用基础油稀释到1%）。
#4 [替代方案]：薰衣草对你无感？试试佛手柑+罗马洋甘菊，或岩兰草。
#5 [配合动作]：扩香的同时做3次深呼吸——鼻吸4秒、屏息4秒、口呼6秒。
```

**问题**
```
#1：你有没有想过，失眠的问题可能根本不在"睡觉"上？
#2：每天晚上，你有没有一个固定的"告诉大脑该休息了"的信号？
#3：你买的精油瓶子上，有写拉丁学名吗？
```

### Step 3: 提供组装方案

```
方案 1：专业权威型
结构：反常识钩子 → 核心洞察 → 类比 → 行动步骤 → 金句
来源：Dan Koe "The Death of Value Content" 的结构变体
调性：专业、清醒、有距离感
适用：建立专家形象、知识分享类内容

方案 2：共情故事型
结构：故事 → 核心洞察 → 类比 → 行动步骤 → 金句
来源：Dan Koe "Fix Your Life in 1 Day" 的情感流动模式
调性：温柔、懂你、像朋友在跟你说话
适用：情感共鸣、生活方式分享

方案 3：实用避坑型
结构：反常识钩子 → 故事（翻车案例） → 数据 → 行动步骤 → 问题
来源：Dan Koe 反共识文章的诊断-解决方案模式
调性：犀利、干货、有"种草拔草"的小红书感
适用：产品推荐、避坑指南

方案 4：身份重构型
结构：问题诊断 → 身份转变 → 行为改变 → 金句
来源：Dan Koe 身份-行为-结果模型的叙事版本
调性：深度、启发性、改变视角
适用：深度内容、公众号文章
```

### Step 4: 用户选择

```
用户：用方案2，但把故事#1换成故事#3

AI：[按自定义混搭组装输出]
```
