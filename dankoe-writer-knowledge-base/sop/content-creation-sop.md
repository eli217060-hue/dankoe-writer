# Dan Koe 内容创作 SOP

> 标准作业流程 - 基于逆向工程 Dan Koe 的写作模式

## SOP-001: 反共识型文章

### 适用场景
- 挑战行业主流观点
- 提供颠覆性视角
- 建立思想领导力

### 标题模式

```markdown
# The Death Of [主流概念]

> [副标题: 具体行动建议]
```

**示例**:
- The Death Of Value-Based Content
- The Death Of Productivity
- The Death Of Thoughtful Creation

### 结构模板

```markdown
# The Death Of [X]

> [行动建议]

## [X] 曾经有效

[历史背景]
[为什么过去成功]

## 为什么现在失效了

[时代变化分析]
[核心矛盾揭示]

## 新的成功路径

[你的反论]
[具体策略]

## 行动清单

- [ ] [第一步]
- [ ] [第二步]
- [ ] [第三步]
```

### 写作要点

1. **开场否定主流**
   - 直接说出大家都相信的东西
   - 然后说"这是错的"

2. **诊断问题根源**
   - 不要停留在表面
   - 深入心理和社会层面

3. **提供新范式**
   - 必须同样简单易懂
   - 但更有解释力

4. **给具体行动**
   - 不能只批判不建设
   - 提供可执行的步骤

## SOP-002: 方法论型文章

### 适用场景
- 分享具体技巧
- 教授可执行方法
- 构建个人系统

### 标题模式

```markdown
# How to [效果] in [时间]

> [承诺/副标题]
```

**示例**:
- How to fix your entire life in 1 day
- How to build a world (the 2-hour content ecosystem)

### 结构模板

```markdown
# How to [X]

> [副标题]

## 为什么大多数人失败

[问题诊断]
[数据支持]

## 核心原理

[你的独特见解]
[反直觉洞察]

## 具体方法

### 第一步: [名称]
[详细说明]

### 第二步: [名称]
[详细说明]

### 第三步: [名称]
[详细说明]

## 常见陷阱

- [陷阱1]: [如何避免]
- [陷阱2]: [如何避免]
```

### 写作要点

1. **先诊断后治疗**
   - 不要直接给方案
   - 先分析为什么失败

2. **提供原理层解释**
   - 不只是"怎么做"
   - 更要"为什么这样做"

3. **分步骤可执行**
   - 每步都是具体行动
   - 可以立即开始

## SOP-003: 规则列表型文章

### 适用场景
- 整合多年经验
- 创建参考清单
- 易于消化和记忆

### 标题模式

```markdown
# [数字] rules to [效果] in [时间]

> [副标题]
```

**示例**:
- 12 rules to change your life in 12 months
- 7 ideas that will change your life

### 结构模板

```markdown
# [N] Rules to [X]

> [副标题]

## Rule 1: [核心主张]

[解释或故事]
[洞察或例子]

## Rule 2: [核心主张]

[解释或故事]
[洞察或例子]

...

## Rule N: [最深刻的洞察]

[总结和升华]

---

If you only take one thing from this...
[核心信息浓缩]
```

### 规则设计原则

1. **独立性**: 每条规则可单独理解
2. **递进性**: 规则之间有逻辑关系
3. **完整性**: 覆盖主题的关键方面
4. **可执行**: 每条都有行动暗示

### 规则格式

```markdown
## Rule X: [一句话核心主张]

[解释 2-3 句]
[例子或故事 1-2 句]
[反直觉洞察]
[行动暗示]
```

## SOP-004: 钩子与开场

### 开场模式库

#### 模式 1: 数据冲击

```markdown
[惊人数据].
And that's okay.
Most people do.

Studies actually show that [支持数据]...
```

#### 模式 2: 权威引用

```markdown
> [名人名言]

This quote changed everything for me.
Here's why [你的解读]...
```

#### 模式 3: 问题诊断

```markdown
You're probably going to [失败行为].
And that's okay.
Most people do.

The real reason is [深层原因]...
```

#### 模式 4: 反直觉陈述

```markdown
[主流观点] is wrong.

Here's what [权威/研究] actually shows...
```

#### 模式 5: 个人故事

```markdown
I used to believe [旧信念].

Then [转折事件].

Now I understand [新认知]...
```

## SOP-005: 金句模板

### 身份与行为类

```
The [成功者] has to grind to [负面行为].
They can't see themselves living any other way.
```

```
You aren't where you want to be 
because you aren't the person who would be there.
```

```
If you want [结果],
you must be [身份]
long before you reach it.
```

### 目标与感知类

```
A goal is a lens of perception.
You only see what your goals allow you to see.
```

```
Most people set surface-level goals
and wonder why they get surface-level results.
```

### 行动与结果类

```
High intelligence is the ability to iterate and persist.
Low intelligence is getting stuck on problems instead of solving them.
```

```
Every problem can be solved on a large enough time scale.
Your excuses are signs of low intelligence.
```

### 创作与差异类

```
In a world of infinite content,
depth becomes the differentiator.
```

```
Your unique perspective is the final, unreplicable moat.
AI can do everything but see the world as you do.
```

## SOP-008: 引用与权威模式

### 引用格式

```markdown

> — [作者]

[你的解读和扩展]
```

### 多引用模式

```markdown
This isn't just my opinion:

> [引用1]
> — [作者1]

> [引用2]
> — [作者2]

Both point to the same truth: [综合洞察]
```

### Dan Koe 常用权威来源

- **心理学**: Alfred Adler, Maxwell Maltz
- **商业**: Naval Ravikant, Paul Graham
- **历史**: Adam Smith, Leonardo da Vinci

## SOP-010: 内容积木系统

> 详细内容见 [SOP-010-内容积木.md](./SOP-010-内容积木.md)

### 设计原则

```
积木生成：不是模板填空，是学以致用
├── 类型框架：固定8类
├── 具体内容：动态生成
└── 生成依据：对 Dan Koe 文章的分析

组装方案：不是写死的，是分析出来的
├── 来源：Dan Koe 文章结构预分析
├── 数量：3-5个，不固定
└── 标注：结构来源 + 适用场景 + 调性
```

### 积木库架构

```
内容积木库
├── 核心洞察    # 深度观点，文章的灵魂
├── 金句        # 短小精炼，传播利器
├── 故事        # 场景代入，证据支撑
├── 数据        # 科学背书，可信度加成
├── 类比        # 形象比喻，降低理解门槛
├── 反常识钩子  # 开场引爆，打破认知
├── 行动步骤    # 具体方法，立即可做
└── 问题        # 互动引导，评论区杀手
```

### 积木内容来源

从 Dan Koe 文章分析中提取：

| 分析维度 | 应用方式 |
|----------|----------|
| 心理触发机制 | 生成钩子、洞察 |
| 论证逻辑 | 生成核心洞察 |
| 类比方法 | 生成类比积木 |
| 故事结构 | 生成故事积木 |

### 处理流程

```
用户选择角度
     ↓
① 调用 Dan Koe 文章分析（心理机制、论证逻辑、结构模式）
     ↓
② 动态生成积木库（8类，约32个积木）
     ↓
③ 提供 N 个组装方案（基于 Dan 结构分析，标注来源+调性）
     ↓
④ 用户选择：推荐方案 OR 自定义混搭
     ↓
⑤ 组装输出
```
