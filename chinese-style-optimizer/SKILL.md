# 中文文笔优化器 V2

> 基于优质中文范例优化文章文笔
>
> **核心原则：只优化文笔和表达方式，绝不改变原有框架和结构**

---

## 一、定位与分工

### 与 Dankoe Writer 的分工

```
Dankoe Writer 负责（不动）：
├─ 结构设计
├─ 框架选择
├─ 逻辑安排
├─ 节奏控制
└─ 观点输出

本模块负责（只改这些）：
├─ 句式优化：生硬 → 自然
├─ 口语化：书面 → 口语
├─ 情绪表达：平淡 → 有情绪
├─ 画面感：抽象 → 具体
├─ 金句打磨：提炼可传播的句子
└─ 去 AI 味：消除机器感（V2 新增）
```

### 工作流程

```
Dankoe Writer 生成初稿（结构 + 框架已定）
        ↓
Chinese Style Optimizer 优化文笔
        ↓
去 AI 味检测（V2 新增）
        ↓
输出最终版本
```

## 二、文笔优化技巧

### 2.1 去翻译腔

| 生硬表达 | 自然表达 |
|---------|---------| 
| 因此 | 所以嘛 |
| 然而 | 但是呢 |
| 综上所述 | 说白了就是 |
| 由此可见 | 你看 |
| 值得注意的是 | 有个事儿你得知道 |
| 对此我感到 | 我觉得吧 |
| 这一点非常重要 | 这事儿特别关键 |
| 在某种程度上 | 怎么说呢 |
| 不容忽视的是 | 你可能没注意到 |

### 2.2 去书面语

| 书面语 | 口语化 |
|--------|--------|
| 进行了一次思考 | 想了想 |
| 给予了高度评价 | 说特别棒 |
| 呈现出一种趋势 | 变得越来越... |
| 具有十分重要的意义 | 特别重要 |
| 取得了良好的效果 | 效果挺好的 |
| 实现了突破 | 终于搞定了 |

### 2.3 情绪表达

| 平淡 | 有情绪 |
|------|--------|
| 这让人很生气 | 我气得把手机都摔了 |
| 这件事很遗憾 | 这件事让我难过了整整三年 |
| 我很无语 | 我：？？？？ |
| 很焦虑 | 凌晨三点，盯着天花板，脑子里像有一百个标签在转 |

**情绪放大公式**：情绪 + 具体画面 / 极端后果

### 2.4 画面感

| 抽象表达 | 画面表达 |
|---------|---------| 
| 她很焦虑 | 凌晨三点，她盯着天花板 |
| 他很孤独 | 他一个人吃火锅，对面放着包 |
| 生活很苦 | 住在地下室，衣服总有股霉味儿 |

**画面感公式**：具体场景 + 细节 + 情绪

### 2.5 金句打磨

**对比型**：A 不是 X，而是 Y
```
大多数婚姻不是死于无性，而是死于无话
```

**否定型**：X？错。Y。
```
努力就能成功？错。方向比努力重要 100 倍
```

**重定义型**：X 不是 Y，而是 Z
```
睡眠不是任务，是礼物
```

## 三、去 AI 味检测（V2 新增）

优化完成后，自动执行以下检测。**触发任何一项则自动改写对应部分**：

### 3.1 关键词复读检测

```
□ 同一关键词/形容词出现超过 5 次？
  → 用同义词、近义词、或完全不同的表达方式替换
  → 例："明确"出现 30 次 → 替换为"清晰的""具体的""一听就懂的"或直接删掉
```

### 3.2 套路过渡句检测

```
□ 是否出现以下套路？
  - "让我分享一个真实经历"  → 删掉，直接讲故事
  - "最后想说"              → 删掉，直接给最后一句话
  - "接下来我要告诉你"      → 删掉，直接说
  - "在这里我想强调"        → 删掉
  - "你可能会问"            → 如果没必要就删掉
```

### 3.3 格式化检测

```
□ 是否有 bullet point 过多的段落？
  → 短文中不应该有 bullet point，改为自然段落
  → 长文中 bullet point 不超过 2 组

□ emoji 数量是否合适？
  → 小红书：不超过 8 个
  → 公众号/知乎：不超过 3 个
  → Newsletter：0-1 个
```

### 3.4 三段式排比检测

```
□ 是否有 AI 特别喜欢的"三段式排比"？
  例：
  "它不是A，而是B。
   它不是C，而是D。
   它不是E，而是F。"
  → 如果没有实际信息增量，砍掉 2 段只留最好的 1 段
```

### 3.5 结构机械感检测

```
□ 是否有明显的"先总后分"AI 结构？
  例："以下从三个方面来分析..."
  → 改为自然过渡，不要告诉读者你在"分析"
```

## 四、自我学习机制

### 触发方式

| 方式 | 说明 |
|------|------|
| 手动触发 | 用户说"消化范例库""分析学习""进化一下" |

### 学习流程

```
用户添加范例文章
    ↓
分析：提取句式 / 情绪表达 / 画面感 / 金句公式
    ↓
与 wording.md 比对：标记新增 / 去重
    ↓
更新 wording.md
    ↓
输出学习报告
```

### 分析维度

| 维度 | 提取内容 | 存入位置 |
|------|---------|---------| 
| 句式 | 口语化表达、去翻译腔方式 | wording.md |
| 情绪 | 情绪放大、吐槽自嘲技巧 | wording.md |
| 画面感 | 抽象→具体转换方式 | wording.md |
| 金句 | 金句公式 + 示例 | wording.md |

## 七、核心检查清单

优化完成后检查：

**只改了文笔**：
- [ ] 句式是否更自然？
- [ ] 是否去掉了翻译腔？
- [ ] 情绪表达是否更到位？
- [ ] 画面感是否更强？
- [ ] 金句是否更出彩？

**没动结构**：
- [ ] 段落数量一致？
- [ ] 框架位置不变？
- [ ] 论证顺序不变？
- [ ] 核心观点不变？
- [ ] 篇幅相当？

**去 AI 味（V2 新增）**：
- [ ] 关键词没有复读？
- [ ] 没有套路过渡句？
- [ ] 格式没有过度 bullet point？
- [ ] 没有无意义的三段式排比？
- [ ] 没有"先总后分"机械结构？

---

*记住：结构是骨架，文笔是血肉。只优化血肉，不动骨架。*
