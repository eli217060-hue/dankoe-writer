## 一、学习触发机制

### 1.1 自动触发条件

| 触发条件 | 说明 | 行为 |
|----------|------|------|
| 新增 5 篇内容 | 知识库新增文章/视频达到阈值 | 自动触发模式分析 |
| 周定期分析 | 每周固定时间（可配置） | 全面审查知识库 |
| 用户请求 | 用户主动要求分析 | 立即执行分析 |

## 二、晋升机制

### 2.1 晋升标准

一个学习条目需要满足以下条件才能晋升到 SKILL.md：

| 条件 | 要求 | 说明 |
|------|------|------|
| 出现次数 | ≥ 3 次 | 在不同内容中反复出现 |
| 验证效果 | 已验证有效 | 在创作中使用过且效果好 |
| 中文适配 | 已完成 | 已适配到中文表达习惯 |
| 普遍适用 | 适用范围广 | 不局限于特定主题 |

### 2.2 晋升流程

```
发现新模式
    │
    ↓
记录到 patterns/ 或 insights/
    │
    ↓
在创作中试用
    │
    ↓
记录效果（验证次数+1）
    │
    ↓
验证次数 ≥ 3？
    │
    ├─ 否 → 继续验证
    │
    └─ 是 → 效果评估
              │
              ↓
         效果良好？
              │
              ├─ 否 → 降级或废弃
              │
              └─ 是 → 晋升到 SKILL.md
                       │
                       ↓
                  更新版本号
                  记录更新日志
```

### 2.3 晋升目标

根据学习内容类型，晋升到不同位置：

| 内容类型 | 晋升位置 | 示例 |
|----------|----------|------|
| 写作框架 | SKILL.md 第四章 | 新的结构模式 |
| 标题公式 | SKILL.md 第四章 | 新的标题类型 |
| 钩子技巧 | patterns/hooks.md | 新的开头方式 |
| 金句模板 | insights/GOLDEN-PHRASES.md | 新的金句公式 |
| 中文适配 | SKILL.md 第四章 | 新的本地化规则 |

## 三、与 self-improving-agent 协同

### 3.1 分工说明

| 本 skill 负责 | self-improving-agent 负责 |
|---------------|---------------------------|
| 内容分析和模式提取 | 错误记录和追踪 |
| 写作知识库管理 | 用户反馈收集 |
| 定期学习报告 | 晋升决策建议 |
| 中文适配优化 | 行为模式调整 |

### 3.2 协同工作流

```
┌─────────────────────────────────────────────────────────────┐
│                     用户使用场景                             │
└─────────────────────────────────────────────────────────────┘
                           │
           ┌───────────────┴───────────────┐
           ↓                               ↓
┌─────────────────────┐         ┌─────────────────────┐
│ 本 skill            │         │ self-improving-agent│
│                     │         │                     │
│ 分析：              │         │ 记录：              │
│ - 哪些模式效果好    │         │ - 用户偏好          │
│ - 哪些结构被选中    │         │ - 常见错误          │
│ - 新发现的技巧      │         │ - 改进建议          │
└─────────┬───────────┘         └─────────┬───────────┘
          │                               │
          └───────────────┬───────────────┘
                          ↓
                ┌─────────────────┐
                │ 联合学习报告    │
                │                 │
                │ 模式洞察 +      │
                │ 用户偏好 =      │
                │ 优化建议        │
                └────────┬────────┘

                ┌─────────────────┐
                │ 执行优化        │

                │ - 调整模式库    │

### 3.3 数据共享

两个 skill 可以共享以下数据：

```

│   └── ...
│

## 五、配置选项

### 5.1 可调整参数

```yaml
# 自我提升配置
self_improve:
  # 触发条件
  triggers:
    content_count: 5          # 新增N篇内容时触发分析
    schedule: "0 9 * * 1"     # 每周一早9点

  # 晋升标准
  promotion:
    min_occurrences: 3        # 最少出现次数
    min_effectiveness: 0.7    # 最低效果评分
    require_chinese: true     # 要求中文适配完成

  # 保留策略
  retention:
    temp_memory_days: 30      # 临时记忆保留天数
    max_patterns: 100         # 最大模式数量
    cleanup_interval: 90      # 清理周期（天）
```

### 5.2 用户可自定义

用户可以在 AGENTS.md 中覆盖默认配置：

```markdown
## Dan Koe Writer 配置

### 自我提升设置
- 自动分析频率：每周
- 晋升阈值：出现3次
- 中文适配：必须

### 偏好设置
- 偏好框架：PASTOR
- 偏好风格：反直觉型
- 常用平台：微信公众号
```

## 六、常见问题

### Q1: 如何手动触发分析？

```
用户：分析知识库，看看有什么新发现
```

### Q2: 如何查看学习进度？

```
用户：显示知识库状态
用户：显示最近的 5 条学习
```

### Q3: 如何删除不需要的学习？

```
用户：删除学习条目 [LRN-XXXXXX-XXX]
```

### Q4: 如何暂停自动学习？

在 AGENTS.md 中添加：
```markdown
## Dan Koe Writer 配置
self_improve_enabled: false
```
