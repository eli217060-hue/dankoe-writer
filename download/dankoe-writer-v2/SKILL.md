# Dan Koe 写作风格 Skill V2

> 基于 Dan Koe 的写作方法论，支持智能分析和自我提升
>
> **核心特点**：
> - 交互式工作流：分析 → 提供选项 → 用户选择 → 生成
> - 知识库系统：持续添加新文章/视频内容
> - 自我提升：定期分析新内容，自动优化写作能力

---

## 四、核心写作方法论

### 4.1 五大写作框架

#### 框架 1：PASTOR（深度长文）

```
P - Problem（问题）
  开篇点出读者面临的具体痛点
  例："你是不是每天忙得要死，却感觉一事无成？"

A - Amplify（放大）
  放大问题的后果和情绪
  例："这种状态持续下去，你会越来越焦虑..."

S - Story/Solution（故事/方案）
  分享你的经历或给出解决方案框架

T - Testimony/Transformation（证据/转变）
  具体案例或数据支撑

O - Offer（提供）
  给出具体的行动框架或步骤

R - Response（回应）
  明确的下一步行动号召
```

#### 框架 2：AIDA（短内容）

```
A - Attention（注意）：用一句震惊的话抓住眼球
I - Interest（兴趣）：用一个反直觉观点引发好奇
D - Desire（欲望）：展示转变后的美好图景
A - Action（行动）：给出一个简单可执行的第一步
```

#### 框架 3：PAS（说服力）

```
P - Problem（问题）：描述具体、可感知的痛点
A - Agitate（煽动）：为什么这个问题一直存在？
S - Solution（解决）：你的独特解决方案
```

#### 框架 4：3问深挖法

```
对任何话题问三个问题：
1. WHAT（是什么？）- 这件事的本质是什么？
2. HOW（怎么做？）- 具体的执行步骤是什么？
3. WHY（为什么？）- 这件事为什么重要？

持续追问"为什么"，直到触及本质
```

#### 框架 5：反共识结构

```
开头：提出一个反直觉的观点
主体：三个支撑论据
  - 为什么常规方法会失败
  - 真实案例或数据
  - 更好的替代方案
结尾：具体行动建议
```

### 4.2 标题公式库

| 类型 | 公式 | 示例 |
|------|------|------|
| 数字型 | [X]个方法让你[结果] | 5个方法让你摆脱内耗 |
| 反直觉 | 为什么[常见观点]是错的 | 为什么努力工作正在毁掉你 |
| 好奇心 | 关于[主题]，没人告诉你的真相 | 关于副业，没人告诉你的真相 |
| 故事型 | 我如何在[时间]内[成就] | 我如何在6个月内做到月入10万 |
| 警告型 | 别再[常见行为]了 | 别再每天制定计划了 |

### 4.3 中文适配规则

```
【句式特点】
- 短句为主，偶尔用长句调节节奏
- 直接问读者："你有没有想过..."
- 使用口语化表达

【金句公式】
- 对比法："大多数人追求[常量]，高手却在做[变量]"
- 否定法："[常见观点]？错。[真实原因]"
- 重新定义："[概念]不是[A]，而是[B]"
- 递进法："不仅...更要..."

【平台适配】
微信：标题要有"利益感"，每300字配图
小红书：标题要有emoji和数字，正文500-800字
即刻：开门见山，更个人化
知乎：问题形式标题，先给结论
```
