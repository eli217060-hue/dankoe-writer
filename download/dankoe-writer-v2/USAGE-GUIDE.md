## 一、安装步骤

### 1.1 下载并安装

```bash
# 步骤1：克隆仓库
git clone https://github.com/eli217060-hue/dankoe-writer.git

# 步骤2：安装到 OpenClaw
cp -r dankoe-writer ~/.openclaw/skills/dankoe-writer

# 步骤3：创建知识库目录
mkdir -p ~/.openclaw/workspace/.dankoe-knowledge/{articles,videos,patterns,insights}

# 步骤4：重启 OpenClaw（如果需要）
```

### 1.2 验证安装

在 OpenClaw 中发送：
```
你有哪些写作相关的技能？
```

如果安装成功，AI 会提到 Dan Koe Writer 技能。

## 二、快速开始

### 2.1 最简单的使用方式

```
我想写关于"拖延症"的内容，给我3个方案
```

AI 会返回 3 个选题方案，包含：
- 标题
- 写作框架
- 核心观点
- 切入角度
- 预估字数

### 2.2 选择并生成

```
选A
```

或者带调整：

```
选A，标题改成更口语化的
```

AI 会根据你的选择生成完整文章。

## 三、核心工作流

### 3.1 标准流程图

```
┌─────────────────────────────────────────────────────┐
│  第1步：告诉 AI 你想写什么                           │
│  例：我想写关于"效率"的内容                         │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  第2步：AI 分析并返回多个方案                       │
│  你会看到 2-5 个不同角度的选题方案                  │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  第3步：选择你喜欢的方案                            │
│  例：选A / 选B，标题改短一点                        │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  第4步：AI 生成完整文章                             │
│  包含标题、正文、金句、行动号召                     │
└─────────────────────────────────────────────────────┘
```

### 3.2 详细示例

**输入**：
```
我想写关于"副业"的内容

给我 3 个选题方案

**AI 返回**：
```

## 四、知识库管理

### 4.1 知识库是什么？

知识库是用来存储 Dan Koe 文章和学习内容的仓库。AI 会从知识库中学习写作技巧，随着你添加更多内容，AI 的写作能力会越来越强。

### 4.2 添加新文章

**方式一：URL 自动抓取（推荐）**

```
学习这篇文章：https://thedankoe.com/xxxxx
```

AI 会自动：
1. 抓取文章内容
2. 分析写作结构
3. 提取可复用模式
4. 存入知识库

**方式二：直接粘贴内容**

```
我有一篇 Dan Koe 的新文章，请学习：

[粘贴文章内容]
```

**方式三：分析视频**

```
分析这个 Dan Koe 的视频：
https://www.youtube.com/watch?v=xxxxx
```

### 4.3 查看知识库状态

```
显示知识库状态
```

返回：
```
知识库状态：
- 文章总数：5
- 视频总数：2
- 提取模式：12
- 金句数量：28
- 最近学习：[文章标题]
```

### 4.4 手动添加文件

如果你想手动管理文件：

```bash
# 文件存放位置
~/.openclaw/workspace/.dankoe-knowledge/articles/

# 按月份组织
mkdir -p ~/.openclaw/workspace/.dankoe-knowledge/articles/2024-01

# 创建文章文件
nano ~/.openclaw/workspace/.dankoe-knowledge/articles/2024-01/article-001.md
```

文件格式：
```markdown
## [KB-202401-001] 文章标题

**来源**: https://thedankoe.com/xxxxx
**添加日期**: 2024-01-15

### 原文内容
[粘贴完整文章内容]
```

## 五、自我提升机制

### 5.1 自动学习

知识库新增 **5 篇内容**时，AI 会自动触发分析：
- 发现新的写作模式
- 提取高频技巧
- 建议更新核心技能

### 5.2 手动触发分析

```
分析知识库，看看有什么新发现
```

AI 会返回学习报告：
```
自我分析报告 - 2024-01-15

一、知识库状态
- 文章总数：10
- 提取模式：15

二、模式发现
- 新发现的标题公式：[列表]
- 新发现的钩子技巧：[列表]

三、建议更新
以下模式已出现 3 次以上，建议整合：
- [具体模式]

四、下一步行动
- [ ] 更新写作框架库
- [ ] 添加新的标题公式
```

### 5.3 晋升机制

当一个模式：
- 出现 3 次以上
- 验证效果良好
- 已适配中文

会被整合到核心技能中，自动提升写作能力。

## 六、高级功能

### 6.1 批量主题分析

不确定写什么？让 AI 帮你分析：

```
分析这些主题，看哪个最适合写：
1. 拖延症
2. 时间管理
3. 精力管理
4. 习惯养成
```

AI 会返回分析报告，包含热度、竞争度、推荐指数。

### 6.2 系列内容规划

```
我想写一个关于"个人成长"的系列，帮我规划5篇内容
```

AI 会规划完整的系列文章，包含每篇的标题、框架、目的。

### 6.3 平台适配

```
把这篇文章改写成小红书版本
```

AI 会自动调整：
- 标题风格
- 字数
- 排版
- 表情符号
- 标签建议

### 6.4 指定写作框架

```
用 PASTOR 框架写一篇关于"专注力"的文章
字数：1500字
平台：微信公众号
```

可用的框架：
- **PASTOR**：深度长文
- **AIDA**：短内容
- **PAS**：说服型内容
- **3问深挖法**：深度分析
- **反共识结构**：爆款内容

## 七、常用命令速查

## 八、常见问题

### Q1: 安装后不生效？

检查：
1. 文件是否放在正确位置：`~/.openclaw/skills/dankoe-writer/SKILL.md`
2. 是否重启了 OpenClaw
3. 文件名是否正确（必须是 `SKILL.md`，大写）

### Q2: 如何让 AI 更好地理解我的需求？

提供更多信息：
```
我想写关于"拖延症"的内容
目标平台：微信公众号
读者画像：25-35岁的上班族
核心观点：拖延不是懒惰，是恐惧
给我 3 个方案
```

### Q3: 生成的内容太长/太短？

指定字数：
```
选A，字数控制在1000字左右
```

或者在初始请求中说明：
```
给我3个方案，每个方案800-1000字
```

### Q4: 如何让 AI 学习我的写作风格？

把你的文章发给 AI：
```
这是我的文章，请学习我的写作风格：

[粘贴你的文章]

然后写一篇关于"新主题"的文章，保持我的风格
```

### Q5: 知识库文件放在哪里？

```
~/.openclaw/workspace/.dankoe-knowledge/
├── articles/      # 文章
├── videos/        # 视频内容
├── patterns/      # 模式库（AI维护）
└── insights/      # 洞察库（AI维护）
```

### Q6: 如何与 self-improving-agent 配合使用？

同时安装 [self-improving-agent](https://clawhub.ai/pskoett/self-improving-agent)：
- Dan Koe Writer：负责内容分析和写作
- self-improving-agent：负责记录错误和用户反馈

两者会自动协同工作。

### Q7: 如何临时禁用某个功能？

在对话中说明：
```
这次不用反共识框架，用方法论型
```

### Q8: 生成的内容不够"Dan Koe 风格"？

1. 添加更多 Dan Koe 的文章到知识库
2. 明确要求：
```
用 Dan Koe 风格写，要有反直觉观点和金句
```
3. 触发学习分析：
```
分析知识库，强化 Dan Koe 风格特征
```

## 附录：写作框架详解

### PASTOR 框架（适合深度长文）

```
P - Problem：开篇点出痛点
A - Amplify：放大问题的后果
S - Story/Solution：故事或解决方案
T - Testimony：证据或转变
O - Offer：具体行动框架
R - Response：行动号召
```

### AIDA 框架（适合短内容）

```
A - Attention：抓住注意力
I - Interest：引发兴趣
D - Desire：激发欲望
A - Action：引导行动
```

### 反共识结构（适合爆款内容）

```
开头：提出反直觉观点
主体：三个支撑论据
结尾：具体行动建议
```
