# 工作流详细说明

详细描述分析→选择→生成的完整流程。

---

## 一、标准工作流

### 阶段 1：需求输入

**用户输入示例**：
```
我想写关于"拖延症"的内容
目标平台：微信公众号
给我 3 个选题方案
```

**AI 处理逻辑**：
1. 解析主题关键词
2. 检索知识库相关模式
3. 分析主题可能的切入角度
4. 匹配合适的写作框架

---

### 阶段 2：智能分析

#### 2.1 主题分析

```
主题：拖延症
├─ 常见认知：拖延是意志力薄弱的表现
├─ 反共识角度：拖延可能是完美主义的副作用
├─ 方法论角度：如何科学地克服拖延
└─ 深度分析角度：拖延背后的心理机制
```

#### 2.2 知识库检索

```
检索结果：
├─ 相关钩子模式：痛点触发型、反直觉型
├─ 相关结构：PASTOR、反共识结构
├─ 相关金句："你不是在拖延，你是在害怕"
└─ 历史成功案例：[LRN-XXXXXX-XXX]
```

#### 2.3 方案生成

为每个方案计算：
- 话题热度
- 反直觉程度
- 实用价值
- 中文适配难度
- 爆款潜力

---

### 阶段 3：提供选项

**标准输出格式**：

```markdown
已分析主题"[主题]"，结合知识库，为你生成 [N] 个选题方案：

---

【方案 A】[类型标签]

**标题**：《[标题]》
**框架**：[框架名称]
**核心观点**：[一句话概括]
**切入角度**：[描述]
**预估字数**：[X]字
**亮点**：[为什么选择这个角度]

**大纲预览**：
1. 开头钩子：[描述]
2. 核心论证：[描述]
3. 行动号召：[描述]

---

【方案 B】[类型标签]
...

---

请选择一个方案（A/B/C），或告诉我需要如何调整：
- 修改标题
- 更换框架
- 调整切入角度
- 混合多个方案
```

---

### 阶段 4：用户交互

#### 4.1 选择方案

**用户**：选 A

**AI**：好的，开始生成方案 A 的完整文章...

#### 4.2 请求调整

**用户**：选 A，但标题改成更口语化的

**AI**：
```
好的，标题优化为：《[新标题]》

正在生成完整文章...
```

#### 4.3 混合方案

**用户**：我喜欢 A 的标题，但用 B 的框架

**AI**：
```
好的，结合方案 A 的标题和方案 B 的框架：

标题：《[A的标题]》
框架：[B的框架]

正在生成完整文章...
```

---

### 阶段 5：内容生成

#### 5.1 生成标准

每篇生成的文章必须满足：

| 要求 | 说明 |
|------|------|
| 框架遵循 | 严格按照选择的框架结构 |
| 钩子有力 | 前50字抓住注意力 |
| 金句嵌入 | 至少包含2-3个可传播的金句 |
| 行动明确 | 结尾有具体的行动号召 |
| 中文自然 | 符合中文阅读习惯 |

#### 5.2 质量检查

生成后自动检查：
```
□ 标题是否吸引人？
□ 开头是否有强钩子？
□ 是否有反直觉观点？
□ 是否有个人故事或案例？
□ 是否有可执行的行动步骤？
□ 是否有金句？
□ 结尾是否有明确的CTA？
□ 字数是否在预期范围？
```

## 二、高级工作流

### 2.1 批量主题分析

**用户输入**：
```
分析这些主题，看哪个最适合写：
1. 拖延症
2. 时间管理
3. 效率提升
4. 精力管理
```

**AI 输出**：
```
主题分析报告：

| 主题 | 热度 | 竞争 | 反直觉空间 | 推荐指数 |
|------|------|------|------------|----------|
| 拖延症 | ⭐⭐⭐⭐ | 高 | 高 | ⭐⭐⭐⭐ |
| 时间管理 | ⭐⭐⭐ | 高 | 低 | ⭐⭐⭐ |
| 效率提升 | ⭐⭐⭐⭐ | 中 | 中 | ⭐⭐⭐⭐ |
| 精力管理 | ⭐⭐⭐ | 低 | 高 | ⭐⭐⭐⭐⭐ |

推荐顺序：精力管理 > 拖延症 > 效率提升 > 时间管理

建议选择"精力管理"，因为：
- 竞争较小，容易脱颖而出
- 有较大的反直觉空间
- 与"效率"和"时间管理"形成差异化
```

### 2.2 系列内容规划

**用户输入**：
```
我想写一个关于"副业"的系列，帮我规划5篇内容
```

**AI 输出**：
```
"副业"系列内容规划：

【第1篇】认知篇
标题：《为什么你的副业一直做不起来？》
框架：反共识结构
目的：打破错误认知，建立信任

【第2篇】方法篇
标题：《从0到月入1万，我只用了这3个步骤》
框架：PASTOR
目的：给出具体方法，建立权威

【第3篇】案例篇
标题：《那些辞职做副业的人，后来都怎么样了？》
框架：故事先行型 PASTOR
目的：案例验证，增强信心

【第4篇】工具篇
标题：《我用这5个工具，把副业效率提高了10倍》
框架：数字型 + 方法论
目的：提供实用价值

【第5篇】进阶篇
标题：《副业月入5万后，我才明白的3个真相》
框架：反共识结构
目的：深化认知，引发行动

是否按此规划开始创作？建议从第1篇开始。
```

### 2.3 平台适配

**用户输入**：
```
把这篇文章改写成小红书版本
```

**AI 处理**：
```
小红书适配版本：

【标题】
原：为什么你的副业一直做不起来
改：副业做不起来？99%的人都踩了这3个坑 💼

【正文结构】
- 字数控制在600字以内
- 每100字分段
- 加入emoji增加视觉吸引力
- 关键信息用【】标注

【配图建议】
图1：封面 - 大字标题 + 冲击性数据
图2-4：核心要点拆解
图5-6：行动步骤
最后：关注引导

【标签建议】
#副业 #搞钱 #上班族 #创业思维

【适配后正文】
[小红书版本的完整内容]
```

## 三、错误处理

### 3.1 主题不明确

**用户**：我想写点什么

**AI**：
```
需要更多信息来帮你分析：

请告诉我：
1. 你想写什么主题？（比如：成长、赚钱、写作...）
2. 目标平台是什么？（微信、小红书、即刻...）
3. 有什么特别想表达的观点吗？

或者，我可以推荐一些当前热门的主题供你选择。
```

### 3.2 选择无效

**用户**：选 D

**AI**：
```
抱歉，我只提供了 A/B/C 三个方案。

请选择：
- A：[简要回顾]
- B：[简要回顾]
- C：[简要回顾]

或者告诉我你需要什么样的调整。
```

### 3.3 调整无法满足

**用户**：选 A，但要像 B 一样深度，像 C 一样有趣

**AI**：
```
我理解你想要：
- A 的整体框架
- B 的深度分析
- C 的趣味性

这是一个很好的组合！让我调整一下...

【新方案】
标题：[优化后的标题]
框架：A + 深度元素
风格：增加趣味性

这样调整可以吗？
```

## 四、快捷命令

### 4.1 极简模式

```
写一篇关于[主题]的文章
```
→ AI 自动完成分析→选择→生成，使用最优方案

### 4.2 快速方案

```
给我 [主题] 的 3 个标题选项
```
→ 只返回标题，不生成内容

### 4.3 大纲模式

```
分析 [主题]，只给大纲
```
→ 返回方案和大纲，不生成正文

### 4.4 修改模式

```
把这篇文章改写成 [平台] 风格
```
→ 基于现有内容适配平台
