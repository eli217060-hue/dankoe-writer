## 🎯 创作流程（V3.1 更新）

```
用户提出主题
     ↓
第一步：确认基础框架（身份/平台/篇幅）
     ↓
AI 内部：快速资料扫描
     ↓
第二步：提供 6+ 个切入角度
        （只呈现观点+差异+价值，不标注类型）
     ↓
用户选择角度
     ↓
第三步：AI 自动完成
        ① 深度追问 → 挖到底层规律
        ② 确认细节 → 不需要用户输入
        ③ 生成内容积木 → 核心观点+联系+场景
        ④ 匹配结构 → 观点确定后才选择结构
        ⑤ 生成初稿
     ↓
第四步：提供标题选项
     ↓
用户选择标题
     ↓
完成
```

## 🧱 内容积木系统（V3.2 核心功能）

### 设计原则

```
积木生成：不是模板填空，是学以致用
├── 类型框架：固定8类
├── 具体内容：动态生成
└── 生成依据：对 Dan Koe 文章的分析（心理机制、论证逻辑...）

组装方案：不是写死的，是分析出来的
├── 来源：Dan Koe 文章结构预分析
├── 数量：3-5个，不固定
└── 标注：结构来源 + 适用场景 + 调性
```

### 积木库架构

```
内容积木库
├── 核心洞察    # 深度观点，文章的灵魂
├── 金句        # 短小精炼，传播利器
├── 故事        # 场景代入，证据支撑
├── 数据        # 科学背书，可信度加成
├── 类比        # 形象比喻，降低理解门槛
├── 反常识钩子  # 开场引爆，打破认知
├── 行动步骤    # 具体方法，立即可做
└── 问题        # 互动引导，评论区杀手
```

### 积木内容来源

从 Dan Koe 文章分析中提取：

| 分析维度 | 应用方式 |
|----------|----------|
| 心理触发机制（恐惧、身份危机、反共识...） | 生成钩子、洞察 |
| 论证逻辑（身份-行为-结果、目标镜头...） | 生成核心洞察 |
| 类比方法（电脑关机、日落、办公室...） | 生成类比积木 |
| 故事结构（个人转变、翻车案例...） | 生成故事积木 |

### 用户选择环节

```
生成积木库（8类，约32个积木）
     ↓
提供 N 个组装方案（标注来源+调性）
     ↓
用户选择：
  A. 采用推荐方案之一
  B. 自定义混搭（指定用哪些积木）
     ↓
组装输出
```

## 🔧 核心写作框架

### Dan Koe 核心模式

#### 1. 身份-行为-结果模型

```
行为是身份的外在表现
持久改变需要身份转变，而非行为约束

核心金句：
"You aren't where you want to be 
because you aren't the person who would be there."
```

#### 2. 标题策略

| 策略 | 公式 | 示例 |
|------|------|------|
| 恐惧驱动 | The Death Of [X] | The Death Of Value-Based Content |
| 利益承诺 | How to [X] in [Y] | How to fix your entire life in 1 day |
| 反直觉 | Why [常识] Is Wrong | Why work-life balance will ruin your life |
| 身份挑战 | You Aren't [X] (Yet) | You aren't where you want to be |

#### 3. 开场钩子模式

```markdown
# 模式 1: 数据冲击
[惊人数据].
And that's okay.
Most people do.

# 模式 2: 权威引用
> [名人名言]
This quote changed everything for me.

# 模式 3: 问题诊断
You're probably going to [失败行为].
And that's okay.
The real reason is [深层原因]...

# 模式 4: 反直觉陈述
[主流观点] is wrong.
Here's what actually works...

# 模式 5: 个人故事
I used to believe [旧信念].
Then [转折事件].
Now I understand [新认知]...
```

## 🤝 与 Chinese Style Optimizer 的协作

```
用户：我想写关于[主题]的文章

Dankoe Writer (本 Skill):
  1. 收集基础框架（身份/平台/篇幅）
  2. 提供切入角度选项
  3. 提供标题选项
  4. 确认细节
  5. 匹配 SOP，生成结构化初稿
  6. 传递给 Chinese Style Optimizer

Chinese Style Optimizer:
  1. 接收初稿
  2. 优化文笔（不动结构）
  3. 输出最终版本
```
